계정·BU를 먼저 만들어야 DE를 생성할 BU가 있고, DE가 있어야 Journey에서 발송 대상을 지정할 수 있습니다. 발송 설정(도메인·IP)은 콘텐츠보다 먼저 잡아야 DNS 전파 시간(24~48시간)을 콘텐츠 제작 기간과 병행할 수 있습니다.

📖 핵심 용어 — 처음 보는 약어 정리
BU — Business Unit
비즈니스 유닛
MCE의 독립 운영 단위. Parent BU(전체 관리)와 Child BU(실제 발송)로 나뉩니다. 브랜드별·국가별로 Child BU를 분리합니다.
DE — Data Extension
데이터 익스텐션
구독자·발송 데이터를 담는 테이블(DB의 테이블과 유사). MCE에서 List 대신 DE를 쓰는 것이 현대적 구축 방식입니다.
SAP — Sender Authentication Package
발신자 인증 패키지
Private Domain(브랜드 전용 발신 도메인) + Dedicated IP(전용 발송 IP) + DKIM 인증을 묶은 패키지. 이메일 도달률에 직결됩니다.
SPF / DKIM / DMARC
이메일 인증 DNS 레코드
SPF: 허가된 발송 서버 목록을 DNS에 등록. DKIM: 이메일에 서명을 붙여 위변조 방지. DMARC: SPF·DKIM 실패 시 거부/격리 정책.
FTP — Enhanced FTP
파일 기반 데이터 서버
고객사 DB → MCE로 구독자 파일을 매일 올리거나, MCE가 Automation Studio로 자동 Import할 때 사용하는 파일 서버입니다.
IP Warming
IP 워밍업
신규 전용 IP는 발신 이력이 없어 스팸으로 분류될 수 있습니다. 소량(200건)부터 수주에 걸쳐 점진적으로 발송량을 늘려 ISP 신뢰를 쌓는 과정입니다.
Org
Salesforce 계정 환경
Salesforce(혹은 MCE) 전체 인스턴스를 가리키는 단어. 한 고객사 = 보통 1개 Org. Org 안에 여러 BU가 있습니다.
Send Classification
발송 분류
Commercial(마케팅·뉴스레터)과 Transactional(주문확인·OTP) 두 종류. Transactional은 수신거부자에게도 발송 가능해 법적 구분이 중요합니다.
💡
읽는 순서 추천: 이 페이지 → Step 0(사전 준비) → Step 1(계정/BU 설정) 순으로 진행하세요. 각 Step의 체크리스트에는 마우스를 올리면 용어 설명이 나오는 항목이 있습니다.

💻 SQL — 발송 대상 추출 (수신거부 제외)
SQL (SFMC Query Studio)
SELECT m.ContactKey, m.EmailAddress, m.Name, m.MemberGrade FROM [Master_DE] m -- AllSubscribers 수신거부자 제외 (Commercial 발송 시 필수) LEFT JOIN _Subscribers s ON m.EmailAddress = s.EmailAddress WHERE m.EmailOptIn = 1 AND m.IsActive = 1 AND m.EmailAddress IS NOT NULL AND (s.Status IS NULL OR s.Status = 'Active')
💻 SQL — 최근 30일 미오픈 구독자 추출
SQL (SFMC Query Studio)
-- _Open 시스템 DE 활용 (Tracking 데이터) SELECT DISTINCT m.ContactKey, m.EmailAddress FROM [Master_DE] m WHERE m.EmailOptIn = 1 AND m.IsActive = 1 AND m.ContactKey NOT IN ( SELECT SubscriberKey FROM _Open WHERE EventDate >= DATEADD(DAY, -30, GETDATE()) )
📋 수신동의 → MCE 연동 흐름
📌
MCE는 수신동의 여부를 DE 필드(EmailOptIn)와 AllSubscribers 상태(Status=Active/Unsubscribed) 두 곳에서 이중 관리합니다. 두 값이 불일치하면 발송 누락 또는 의도치 않은 발송이 발생합니다.
연동 방식 흐름 적합 상황 주의사항
API (실시간) 웹폼 동의 → REST API Upsert → DE 즉시 반영 가입·탈퇴 실시간 처리 Installed Package API 키 필요, 에러 핸들링 필수
FTP 배치 CRM 추출 CSV → /Import → Import File Activity → DE Upsert 일 1회 대량 동기화 실시간성 없음, 파일 명명·인코딩 규칙 사전 정의 필요
Synchronized DE SF CRM Contact/Lead → Marketing Cloud Connect → MCE DE 자동 동기화 CRM 기반 B2B, SF CRM 연동 환경 MC Connect 라이선스 필요, 필터 설정 오류 시 전체 레코드 덮어쓰기 위험
🚫
수신거부자 재발송 방지: AllSubscribers Status = Unsubscribed인 주소는 Commercial Send Classification으로 발송 불가합니다. DE의 EmailOptIn만 관리하고 AllSubscribers 연동을 누락하면 발송 오류가 발생합니다.

SELECT ContactKey, EmailAddress, Name, LastPurchaseDate, DATEDIFF(DAY, LastPurchaseDate, GETDATE()) AS DaysSinceLastPurchase FROM [Master_DE] WHERE IsActive = 1 AND EmailOptIn = 1 AND LastPurchaseDate IS NOT NULL AND DATEDIFF(DAY, LastPurchaseDate, GETDATE()) >= 180
💻 SQL — 일별 이메일 발송 성과 집계
SQL (SFMC Query Activity — 시스템 DE 활용)
-- _Sent, _Open, _Click 시스템 DE 활용 SELECT s.JobID, s.EmailName, COUNT(DISTINCT s.SubscriberKey) AS TotalSent, COUNT(DISTINCT o.SubscriberKey) AS TotalOpens, COUNT(DISTINCT c.SubscriberKey) AS TotalClicks, CAST(COUNT(DISTINCT o.SubscriberKey) * 100.0 / NULLIF(COUNT(DISTINCT s.SubscriberKey), 0) AS DECIMAL(5,2)) AS OpenRate FROM _Sent s LEFT JOIN _Open o ON s.JobID = o.JobID AND s.SubscriberKey = o.SubscriberKey LEFT JOIN _Click c ON s.JobID = c.JobID AND s.SubscriberKey = c.SubscriberKey WHERE CAST(s.EventDate AS DATE) = CAST(DATEADD(DAY,-1,GETDATE()) AS DATE) GROUP BY s.JobID, s.EmailName
📌
SFMC SQL 제약사항: INSERT / UPDATE / DELETE 불가 (SELECT만 허용). 임시 테이블(#temp), 저장 프로시저 사용 불가. 날짜 함수는 GETDATE() 사용 (NOW() 아님). DATEADD, DATEDIFF, CONVERT 사용 가능.

SQL (Automation Studio)
