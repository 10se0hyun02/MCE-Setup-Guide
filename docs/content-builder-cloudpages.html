Content Builder & CloudPages

최종 업데이트: 2026.06.19  |  번역: MCE 구축 가이드 팀

개요

Marketing Cloud Engagement의 콘텐츠 도구는 크게 두 가지입니다. CloudPages는 랜딩 페이지·마이크로사이트·폼 등 웹 콘텐츠를 직접 생성·배포하는 도구이고, Content Builder는 이메일·SMS·Push 등 모든 채널에 사용되는 이미지·문서·콘텐츠를 단일 저장소에서 관리하는 크로스채널 콘텐츠 관리 도구입니다.

도구주요 기능주 사용 목적
CloudPages 랜딩 페이지, 마이크로사이트, Smart Capture 폼, 코드 리소스 잠재고객 확보, 이벤트·프로모션 페이지, Journey 진입점 생성
Content Builder 이메일·SMS·Push 콘텐츠, 이미지·파일 저장소, 템플릿, 콘텐츠 블록 채널별 콘텐츠 생성·관리, 재사용 가능한 에셋 운영

CloudPages

CloudPages는 Marketing Cloud Engagement의 장점을 결합하여 잠재고객 데이터를 캡처·정리·활성화합니다. Smart Capture 폼으로 오디언스를 늘리고, 이를 Journey로 연결하여 잠재고객을 육성합니다. 모바일·소셜·웹 경험에 최적화된 콘텐츠를 어디서든 제공하고, 내장 리포트로 성과를 모니터링할 수 있습니다.

CloudPages 활용 사례

  • 프로모션·청탁 캠페인용 랜딩 페이지 설계
  • 신제품·서비스·캠페인용 마이크로사이트 제작
  • Content Builder 블록 재사용으로 일관성 및 효율성 향상
  • 개인화된 동적 콘텐츠 호스팅 (환경설정, 프로필 정보 등)
  • AMPscript를 활용한 로그인 게이팅 및 커스텀 솔루션

권한 설정

CloudPages는 Marketing Cloud Engagement 사용자 권한을 지원합니다. 웹 콘텐츠를 생성·관리하는 사용자에게 개별 권한 또는 역할을 배정할 수 있습니다.

랜딩 페이지 생성

랜딩 페이지는 많은 마케팅 캠페인의 핵심 요소입니다. CloudPages에서 랜딩 페이지는 마이크로사이트의 기본 빌딩 블록이기도 합니다.

📌
Private Domain이 설정된 경우 루트 레벨 페이지와 오류 처리 페이지(404, 500, 503)를 생성할 수 있습니다. 도메인당 각 타입 1개씩 설정 가능합니다.
⚠️
CloudPages 페이지 로드는 30초 후 타임아웃됩니다. 무거운 AMPscript 연산은 최적화가 필요합니다.

기본 랜딩 페이지 생성 단계

1
CloudPages → 폴더 생성 → Create → Add Content → Landing Page
2
Private Domain이 활성화된 경우 URL 설정 정의
3
랜딩 페이지 저장 → 레이아웃 선택
4
에디터에서 블록을 캔버스로 드래그하여 배치, Content·Block Settings 패널에서 설정 (자동 저장됨)
5
필요시 Smart Capture 또는 Code Resource 블록 추가
6
즉시 게시 또는 예약 게시 설정

Personalized URL (PURL) 설정

Private Domain이 활성화된 경우 친화적이고 개인화된 URL을 설정하여 동적 콘텐츠를 제공할 수 있습니다.

1
CloudPages 폴더 뷰에서 대상 랜딩 페이지 선택 → 설정 아이콘 클릭
2
URL 드롭다운에서 도메인 선택 → Site Key 필드에 읽기 쉬운 이름 입력
3
보안 콘텐츠의 경우 HTTPS 토글 활성화 (SSL 인증서 필요)
4
저장

오류 처리 커스텀 페이지 (403/404/500)

1
CloudPages 폴더 선택 → 새 랜딩 페이지 추가
2
URL 드롭다운에서 Private Domain 선택
3
Advanced → Error Content에서 403/404/500 중 선택
4
페이지 저장 → 레이아웃 선택 → 오류 페이지 콘텐츠 작성 → 게시

마이크로사이트 생성

마이크로사이트는 의미 있는 사이트맵으로 구성된 랜딩 페이지 컬렉션입니다.

1
CloudPages 폴더 생성 또는 선택 → Create → Microsite
2
이름과 선택적 설명 입력
3
Private Domain 사용 시 URL 설정 정의
4
Create 클릭
5
캔버스에 페이지 추가 → 썸네일 클릭하여 편집
6
캔버스에 블록을 드래그하여 배치 및 속성 편집
7
페이지 순서 재정렬 필요 시 현재 버전 복사 없이 사이트맵 생성 → 각 페이지 Link to site map 클릭
8
즉시 게시 또는 예약 게시
💡
블록을 저장하여 다른 곳에서 재사용할 수 있습니다. 블록 클릭 → Save Block → Create New로 저장하세요.

Smart Capture 폼

폼을 제출한 고객으로 Journey를 시작하려면 Smart Capture 폼을 설정하고 Journey Builder에서 진입 이벤트로 설정합니다. Smart Capture 폼은 모든 랜딩 페이지에 추가할 수 있으며, 반드시 멀티스텝 Journey에서만 사용 가능합니다.

⚠️
Smart Capture 폼 데이터는 반드시 Sendable 이벤트 소스 데이터 익스텐션에 저장해야 합니다.
1
데이터 익스텐션 생성
  • Create from New, Is Sendable? 체크
  • 각 속성 상세 설정 (나중에 폼에서 숨기기/표시 가능하므로 관련 데이터 포인트 최대한 포함)
  • Send Relationship: Email Address를 Subscriber Key와 연결
2
랜딩 페이지 생성 또는 편집 → Smart Capture 블록 추가
3
Use in Journey Builder 선택
4
Data Target → Data Extension 선택 → 생성한 데이터 익스텐션 선택
5
폼 필드로 표시할 속성 선택
6
필요 시 숨김 필드 추가 (소스명, 관련 ID 등)
  • Add Hidden Attribute → 필드 선택 → 제출할 값 입력 → Done Editing
7
Submit Options 탭에서 확인 메시지 커스터마이징

Journey Builder에서 Smart Capture 폼 사용

1
Smart Capture 폼이 포함된 랜딩 페이지 생성
2
Journey Builder → 멀티스텝 Journey 생성
3
CloudPages 진입 소스 선택 → 진입 소스 타겟으로 드래그
4
진입 소스 아이콘 클릭 → 목록에서 폼 선택 → Next → 요약 검토 → Done
5
다른 활동·컨트롤 드래그하여 Journey 구성
📌
이메일이 포함된 Journey의 경우 활성화 전 이메일의 Content Detective·Validation 단계를 완료해야 합니다. Smart Capture 폼은 Journey당 1회만 사용 가능합니다.

CloudPages 분석

CloudPages는 최근 90일간 페이지 뷰를 추적하는 내장 분석 기능을 제공합니다. 더 상세한 정보는 Analytics Builder에서 리포트를 생성하세요.

제공 리포트: Microsite Tracking / CloudPages Impressions / CloudPages Smart Capture Submissions

1
CloudPages 폴더에서 검토할 콘텐츠 찾기
2
페이지 썸네일 또는 편집 뷰에서 설정 아이콘 → Activity 선택
3
시작·종료 날짜 조정 후 Submit

Content Builder

Content Builder는 이미지·문서·콘텐츠를 단일 위치에 통합하여 Marketing Cloud Engagement 전 채널에서 사용할 수 있는 크로스채널 콘텐츠 관리 도구입니다. 직관적인 드래그&드롭 인터페이스로 기존 편집 도구보다 효율적이며, 콘텐츠 저장소(CMS)·API·콘텐츠 관리 앱·HTML 콘텐츠 에디터를 통합합니다.

에디터 주요 기능

  • HTML 없이 전문적인 이메일 제작 가능
  • 실시간 라이브 미리보기 (Layout 뷰 및 Code View 동시 반영)
  • HTML 자동완성, 태그 매칭, 색상 코딩, 줄 번호
  • Code Snippet 삽입 (Forward to a Friend, Open Tracking 등 공통 링크)
  • 전역 검색·필터링, 태그·캠페인 연결

관리 설정 (Content Builder Settings)

마케팅 클라우드 툴바 → 사용자명 호버 → Setup → Content Builder Settings

설정 항목기능기본값
Email Autosave이메일 편집 시 변경사항 자동 저장활성화
Across Enterprise Sharing모든 비즈니스 유닛과 콘텐츠 공유비활성화
Predictable File URLs파일명 기반 친화적 URL 생성 (기본: 랜덤 숫자)비활성화
⚠️
Predictable File URLs 활성화 후 이미지를 교체할 때는 Delete가 아닌 Replace를 사용하세요. Delete는 Content Builder 라이브러리에서 파일을 제거하지만 CDN에는 계속 게시된 상태로 남습니다.

Customer Key

각 Content Builder 에셋에는 AMPscript에서 참조할 수 있는 고유 36자 식별자인 Customer Key가 있습니다.

💡
네이밍 규칙 권장: Customer Key는 엔터프라이즈 전체 비즈니스 유닛에서 고유해야 합니다. 대규모 엔터프라이즈에서는 중복 방지를 위해 네이밍 컨벤션을 사용하세요. 예: bu1-hero-logo, bu2-hero-logo. 이름 대신 Customer Key로 참조하면 콘텐츠가 이동되거나 이름이 변경되어도 오류가 발생하지 않습니다.

파일 업로드

파일을 드래그&드롭, 컴퓨터 탐색, 외부 URL을 통해 업로드할 수 있습니다. 파일당 최대 크기: 5MB

문서이미지오디오비디오
.vcf .jpg, .png, .gif, .jpeg .mp3, .aif, .wav .mp4, .avi, .mpeg
📌
커스텀 폰트 파일은 지원하지 않습니다. .htm, .html, .css, .js, .json 등 코드 파일은 서드파티 CDN에 게시되지 않습니다.
📌
Content Builder에 업로드된 대부분의 콘텐츠는 전 세계 최적화 배포를 위해 서드파티 CDN에 저장·게시됩니다. CDN의 공개 URL은 로그인 없이 접근 가능합니다. 게시 완료까지 수분이 소요될 수 있습니다.

파일 업로드 방법

1
Create → Upload → Content
2
Upload 클릭 후 파일 드래그 또는 탐색하여 선택
3
Upload and Publish 클릭

파일 교체 방법

1
해당 파일의 액션 드롭다운 클릭 → Replace
2
교체할 파일 드래그 또는 탐색
3
Upload and Publish

이메일 생성

이메일 이름은 폴더 내에서 고유해야 합니다. 생성 방법은 세 가지입니다.

방법설명사용 시나리오
Template Based기존 템플릿 기반 이메일 생성브랜드 가이드라인이 있는 표준 이메일
HTML PasteHTML 코드 직접 입력 또는 붙여넣기커스텀 디자인 또는 외부 제작 이메일
Text Only텍스트만 포함하는 이메일오픈율 추적 불필요한 단순 알림

Content Builder 이메일 생성 단계

1
Content Builder → Create → Email Message
2
생성 방법 선택 → 필드 입력 → Next
3
제목(Subject) 및 프리헤더 입력 (개인화 문자열·동적 콘텐츠 사용 가능)
4
이메일 콘텐츠 구성 → Next
5
구독자 미리보기·테스트 발송 완료 → 저장
6
발송: Journey Builder 또는 Email Studio 사용
⚠️
이메일 링크에 이메일 주소 등 개인식별정보(PII)를 포함하지 마세요.

템플릿 생성 및 관리

Content Builder에서 HTML 붙여넣기 또는 레이아웃 도구를 사용해 템플릿을 생성합니다.

콘텐츠 영역 슬롯 정의 (HTML Paste 템플릿)

HTML — 슬롯(content area) 정의 예시
<div data-type="slot" data-key="Hero1" data-max-blocks="1" data-allowed-blocks="imageblock; textblock; freeformblock" data-label="여기에 히어로 이미지 배치"> </div>
🚫
예약 키워드 사용 금지 (data-key 값으로 사용 불가): Content, Subcontent, Supercontent, Design, Tags, Filename, Name, legacyID, legacycategoryID, Legacytype, enterpriseID

헤더·푸터 추가 방법 (HTML 템플릿)

HTML — 헤더/푸터 태그
<custom type="header" /> <!-- 헤더 위치 --> <!-- 이메일 본문 --> <custom type="footer" /> <!-- 푸터 위치 --> <!-- 텍스트 전용 이메일의 경우 --> {{> _header}} {{> _footer}}

모바일 반응형 컬럼 스택 처리

CSS — 모바일 미디어 쿼리
@media only screen and (max-width: 480px) { body .responsive-td { width: 100% !important; float: left !important; padding: 0 !important; } }

Einstein Generative AI 콘텐츠 생성

Content Builder에서 Einstein AI를 사용해 제목과 본문을 생성합니다. Einstein Copy Insights에서 작성한 제목/본문을 복사하여 사용할 수도 있습니다.

1
핵심 메시지 입력 (최대 300자)
2
브랜드 퍼소나 선택 (예: 장난스러운, 모험적인 등)
3
선택적으로 샘플 제목 또는 본문 입력
4
생성된 옵션 검토 → 마음에 들지 않으면 재생성
5
이미지는 Typeface 서드파티 연동 사용 (별도 Typeface 계정 필요)

콘텐츠 블록 유형

AMPscript로 Content Builder 블록 참조

AMPscript — 콘텐츠 블록 참조 함수
%%=ContentBlockbyID("12345")=%% %%=ContentBlockbyKey("my-header-block")=%% %%=ContentImagebyID("67890")=%% %%=ContentImagebyKey("logo-image")=%%
📌
AMPscript로 콘텐츠 블록을 가져올 때 에디터가 테이블 래퍼를 추가합니다. 래퍼 없이 코드를 삽입하려면 HTML 블록 대신 Code Snippet을 사용하세요.

AMP for Email

AMP for Email은 구독자가 받은편지함에서 직접 인터랙션하고 오픈 시 실시간 콘텐츠를 제공하는 오픈소스 이메일 표준입니다.

주요 사용 사례

  • 이미지 캐러셀
  • 확인 및 옵트인
  • 실시간 가격·재고 데이터
  • 탐색·리서치 인터랙션

AMP for Email 모범 사례

  • AMP 미지원 클라이언트를 위한 HTML 폴백을 반드시 포함
  • 세 가지 MIME 타입(AMP, HTML, 텍스트) 모두 콘텐츠 작성
  • 제목과 프리헤더는 세 MIME 타입 공통으로 1개
  • AMP 유효성 검사기 AND 이메일 클라이언트 양쪽에서 테스트
  • Multipart MIME 활성화 필수 (Setup → Content Builder Settings)

GTL과 AMP Mustache 충돌 방지

GTL(Guide Template Language)과 AMP Mustache는 동일한 구분자({{ }})를 사용합니다. 충돌 방지를 위해 AMP 이메일 상단에 구분자 변경 코드를 추가하세요.
