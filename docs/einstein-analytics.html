Einstein & Analytics in MCE

최종 업데이트: 2026.06.19 · 출처: Salesforce 공식 문서 (454페이지)

📊 개요

Marketing Cloud Engagement는 Analytics Builder, Collect, Intelligence Reports, Google Analytics 연동, 그리고 다양한 Einstein AI 기능을 통해 캠페인 성과를 측정하고 마케팅 의사결정을 개선합니다.

📈 Analytics Builder

Analytics Builder는 MCE 내 통합 리포팅 도구입니다. 채널별 성과 데이터를 리포트와 대시보드 형태로 조회하고, 예약 발송과 이메일 수신을 설정할 수 있습니다.

주요 기능

  • 이메일 캠페인 참여율·성과 측정
  • 유연한 테이블·차트·그래프로 대시보드 구성
  • Collect 데이터로 클릭·구매 등 연락처 행동 조회
  • 예측 행동 모델 기반 타겟 오디언스 생성
  • Journey Builder 내 단계별 인터랙션 추적 및 목표 달성도 모니터링
⚠️
리포트 파일 크기 제한이메일 발송 리포트는 최대 6MB 이하여야 합니다. 6MB 초과 시 Enhanced FTP로 전송하거나 기간을 분할해 여러 리포트를 생성하세요.
ℹ️
삭제된 사용자의 리포트리포트를 생성한 사용자가 비활성화(삭제)되면 해당 사용자가 만든 리포트도 함께 삭제됩니다. 중요 리포트는 공유 폴더에 저장하거나 예약 실행으로 구성해두세요.

👥 연락처 수 리포트 (Contact Count)

과금의 기준이 되는 연락처 수를 확인하는 리포트입니다. 주 1회 자동 실행되며, 월말에는 월간 보고서가 별도로 생성됩니다.

ℹ️
조회 권한 요건연락처 수 리포트를 보려면 Reports 권한 노드의 연락처(Contacts) 섹션에서 접근(Access) 권한이 활성화되어 있어야 합니다.

🍪 Collect 추적 코드

Collect는 웹사이트에 삽입하는 JavaScript 스니펫으로, 알려진(known) 방문자와 익명(unknown) 방문자 모두의 행동 데이터를 수집합니다. 이 데이터를 통해 연락처 프로필을 보강하고, 예측 추천 및 Journey 최적화에 활용합니다.

Collect로 수집 가능한 데이터

  • 페이지 조회 — URL, 체류 시간, 조회 순서
  • 상품 조회 — 상품 ID, 카테고리, 가격
  • 장바구니 추가/제거
  • 구매 완료 — 주문 ID, 금액, 상품 목록
  • 이메일 클릭 연결 — 이메일 클릭 후 웹 방문 연계 추적

활용 시나리오

  • 장바구니 이탈 Journey — 장바구니 이탈 이벤트를 감지하여 Journey Builder API Entry 트리거
  • 상품 추천 이메일 — 조회 이력 기반 맞춤 상품 추천 이메일 발송
  • 고객 등급 갱신 — 구매 금액 누적 기반 세그먼트 자동 업데이트
  • 이메일 매출 추적 — 이메일 클릭 → 구매까지의 전환 매출 귀속
💡
Collect vs Interactions SDKCollect는 MCE 전용 추적 스크립트입니다. Marketing Cloud Next(Salesforce Personalization)를 사용하는 경우 Interactions SDK로 대체됩니다. MCE만 사용하는 환경에서는 Collect가 적합합니다.

🔗 Google Analytics 연동

Google Analytics를 MCE와 연동하면 Journey 내 고객 행동을 Google Analytics 계정에서도 함께 추적하고, 그 결과를 MCE 화면에서 바로 확인할 수 있습니다.

연동 방식

  • Google 네이티브 인증(OAuth)으로 MCE ↔ GA 보안 연결
  • 이메일 링크에 UTM 파라미터 자동 추가
  • Journey Builder 내 Google Analytics 위젯으로 결과 확인

UTM 파라미터 매핑

UTM 파라미터MCE 자동 입력값
utm_sourcesalesforce.marketing.cloud (기본값)
utm_mediumemail
utm_campaign이메일 발송명
utm_term구독자 키 또는 링크 별칭

🤖 Einstein 기능 개요

MCE의 Einstein은 발송 시간·대상·콘텐츠 최적화에 AI를 활용합니다. 별도 데이터 과학 역량 없이도 데이터 기반 의사결정이 가능합니다.

Send Time Optimization (STO)

개인별 최적 이메일 수신 시간에 자동 발송. 과거 참여 이력 학습.

발송 최적화
Engagement Scoring

이메일 참여 가능성을 0~10 점수로 예측. 높은 점수 = 활성 구독자.

구독자 분류
Content Tagging

이미지·콘텐츠를 AI가 자동 분류·태깅. Content Builder 내 검색 강화.

콘텐츠 관리
Email Subject Line Recommendations

제목 작성 시 과거 성과 데이터 기반 대안 제목 제안.

카피 최적화
Einstein Recommendations

Collect 행동 데이터 기반 개인화된 상품·콘텐츠 추천 이메일 생성.

이메일 개인화
Einstein Frequency

구독자별 최적 이메일 발송 빈도 예측. 발송 과다로 인한 이탈 방지.

피로도 관리
Einstein Splits in Journey

Journey Builder 내 AI 기반 경로 분기. 예측 점수에 따라 자동 분기.

Journey 최적화
Einstein Messaging Insights

캠페인 성과 이상 탐지, 자동 인사이트 알림. 수동 분석 시간 절감.

성과 모니터링

⏰ Einstein Send Time Optimization (STO)

구독자별로 이메일 참여 가능성이 가장 높은 시간을 예측하여 자동으로 최적 시간에 발송합니다.

STO 작동 방식

  1. 데이터 학습 — 각 구독자의 이메일 오픈·클릭 이력(최소 90일) 분석
  2. 예측 모델 생성 — 요일·시간대별 참여 패턴 도출
  3. 발송 큐 배분 — 최적 시간이 도래한 구독자부터 순차 발송 (최대 7일 내)

STO 적용 조건

  • 구독자당 최소 90일, 권장 180일 이상의 이메일 이력 필요
  • 이력이 부족한 구독자는 계정 수준 최적 시간으로 발송
  • Journey Builder 및 Email Studio 단일 발송 모두 지원
💡
STO 사용 시 발송 기간 여유 확보STO는 최대 7일에 걸쳐 발송하므로, 시간 민감한 캠페인(24시간 한정 오퍼 등)에는 적합하지 않습니다. 뉴스레터나 영양가 있는 콘텐츠 발송에 활용하세요.

📉 Einstein Engagement Scoring

각 구독자의 향후 이메일 참여 가능성을 0~10 점수로 예측합니다. 이 점수를 활용해 타겟을 세분화하거나 재참여 캠페인을 설계할 수 있습니다.

점수 구간 활용 전략

점수 구간구독자 상태권장 전략
8–10고활성 (Highly Engaged)신제품 우선 공개, 충성도 프로그램 초대
5–7중간 참여 (Moderate)가치 중심 콘텐츠, A/B 테스트 대상
2–4저참여 (Low Engagement)재참여 캠페인, 발송 빈도 축소
0–1비활성 (Dormant)명시적 재동의 요청, 일정 기간 후 목록 정리 검토

🏷️ Einstein Content Tagging

Content Builder에 업로드한 이미지를 AI가 자동으로 분석하여 콘텐츠 유형·색상·피사체 등의 태그를 부여합니다. 태그 기반 검색으로 에셋 관리가 훨씬 용이해집니다.

자동 태깅 카테고리

  • 피사체 분류 — 인물, 제품, 음식, 자연 환경 등
  • 색상 팔레트 — 주요 색상 자동 추출
  • 감성 분류 — 긍정적, 전문적, 역동적 등 분위기 태그
  • 텍스트 추출 (OCR) — 이미지 내 텍스트 인식 및 검색 인덱싱
ℹ️
태그는 자동 부여되지만 수동으로 수정할 수 있습니다. 브랜드 고유의 분류 체계가 있다면 커스텀 태그를 함께 사용하는 것이 효과적입니다.

🛍️ Einstein Recommendations (이메일 추천)

Collect 추적 코드로 수집된 웹 행동 데이터를 기반으로, 이메일 내 개인화된 상품·콘텐츠를 자동으로 추천합니다.

추천 알고리즘 유형

  • 협업 필터링 — 유사 고객이 조회·구매한 항목 추천
  • 콘텐츠 기반 — 고객이 관심 보인 항목의 속성 유사 항목 추천
  • 인기도 기반 — 전체 방문자 기준 조회·구매 많은 항목

AMPscript 이메일 삽입

Einstein Recommendations 결과를 이메일에 동적으로 삽입하려면 AMPscript의 RecommendationsData() 함수를 사용합니다.

✍️ Einstein 제목 줄 추천

이메일 제목을 입력하면 Einstein이 과거 성과 데이터를 기반으로 클릭율을 높일 수 있는 대안 제목을 제안합니다.

제목 추천 활성화 방법

  1. Email Studio에서 새 이메일 발송 설정 진입
  2. 제목 입력 필드 옆 Einstein 아이콘 클릭
  3. 제안된 제목 목록에서 선택하거나 힌트를 참고해 직접 수정
💡
충분한 이력이 있어야 유효한 추천 가능계정의 이메일 발송 이력이 적을수록 제목 추천의 품질이 낮습니다. 최소 6개월 이상의 발송 이력이 있는 계정에서 가장 효과적입니다.
