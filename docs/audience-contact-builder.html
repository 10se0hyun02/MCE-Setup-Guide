Audience Builder & Contact Builder

Audience Builder로 정교한 세그먼트를 구성하고, Contact Builder로 DE 간 데이터 관계를 설계합니다. Journey Builder의 Decision Split 조건, 발송 대상 세분화 모두 이 두 앱이 기반입니다.
출처: Salesforce 공식 문서 Engagement: Audience Builder and Contact Builder (2026.06.23)

Audience Builder 개요

Audience Builder는 MCE 계정에 저장된 Contact 데이터를 기반으로 동적 대상 그룹을 만드는 앱입니다. 공통 Subscriber Key로 연결된 모든 List, Group, Data Extension의 속성과 행동 데이터를 조합해 정교한 Audience와 Segment를 구성할 수 있습니다.

📌
Enterprise 2.0 전용 기능: Business Unit 필터 등 일부 고급 기능은 Enterprise 2.0 계정에서만 사용 가능합니다. 계정 유형에 따라 사용 가능한 기능이 다를 수 있습니다.

핵심 개념

개념설명비고
Audience Contact 집합. 속성·행동 값의 변화에 따라 동적으로 구성원이 변경됨 Sendable DE와 연결. OMM이 채널·발송 추적에 활용
Segment Audience 내 사용자 정의 기준을 충족하는 하위 그룹 분석 및 타겟팅 용도. Segment 없이 Audience만 사용 가능
Filter Attribute Library 조건으로 Audience 구성원 범위를 좁히는 기능 Filter 없으면 계정 전체 Contact이 대상
Exclusion 특정 조건의 Contact을 Audience/Segment에서 제외하는 기능 Exclusions 탭에서 설정. 구 방식의 Is Not/Not in List 대체
Attribute Library Audience 생성 시 사용할 수 있는 Contact 속성 목록 Contact Builder Attribute Group에서 등록된 속성이 표시됨
Publish Definition 발행된 Audience가 저장되는 DE. Email Studio 등에서 재사용 가능 Audience 데이터 익스텐션 폴더에 저장

Filter 탭 — Audience 범위 설정

1
Audience Builder → Create Audience 클릭 (또는 저장된 Audience 선택)
2
Audience 이름 입력 후 Done 클릭
3
Filter 탭에서 Attribute Library의 항목을 캔버스로 드래그 → 값과 연산자(Equals, Greater than, Range 등) 설정
4
여러 조건 그룹화: 3개 이상 조건 드래그 후 인접 조건 사이의 그룹 아이콘 클릭
5
그룹 연산자 변경: AND / OR / INCLUDE 선택
💡
Subscriber Key 사용 권장: Audience Builder는 Subscriber Key 기반 사용을 권장합니다. 이메일 주소를 키로 사용할 수도 있지만, Subscriber Key를 사용해야 멀티채널 데이터 통합이 정확히 이루어집니다.

Segment 탭 — 하위 그룹 설정

Segment는 Audience 내에서 추가 기준으로 분리한 하위 그룹입니다. 분석 리포팅과 AMPScript 동적 콘텐츠 연동에 활용됩니다.

1
Segment 탭 → Attribute Library에서 조건 드래그 (또는 저장된 Segment Template 사용)
2
Segment 편집: 조건 추가, 레이블 변경, Tracking Code 설정
3
Cap 설정 (선택): 건수 또는 비율로 Segment 크기 제한 — 초과 시 무작위 추출
4
Refresh Counts 클릭으로 건수 확인
📌
AMPScript 연동: Segment 할당 정보를 AMPScript와 SSJS로 조회해 이메일 내 콘텐츠 결정에 활용할 수 있습니다. Contact가 속한 Segment에 따라 다른 배너·쿠폰을 표시하는 패턴에 사용됩니다.

Exclusion 탭 — 발송 제외 조건

특정 속성을 가진 Contact를 Audience 또는 Segment에서 제외합니다. 구 버전의 "Is Not", "Does Not Contain", "Not in List" 연산자를 대체하는 공식 방법입니다.

⚠️
구 연산자 마이그레이션: 기존에 Is Not / Does Not Contain / Not in List 연산자를 사용한 Audience는 여전히 동작하지만, 신규 제외 조건은 반드시 Exclusions 탭을 사용해 재구성하세요.

Audience 발행 (Publish)

Audience를 발행하면 채널별 DE(Publish Definition)가 생성됩니다. Email Studio, Journey Builder 등에서 이 DE를 발송 대상으로 사용할 수 있습니다.

채널앱발행 효과
EmailEmail StudioSendable 이메일 DE 생성. Email Studio 구독자 탭 Audience 폴더에 저장
MobileConnectMobileConnectMobileConnect 내 Contact 리스트 생성
MobilePushMobilePushMobilePush 내 Contact 리스트 생성
Entire Population없음Non-sendable Audience DE 생성. 모든 채널 구독자 포함
SalesforceSales CloudSalesforce Sendable DE 생성. Salesforce Contact/Lead ID가 있는 구독자만 트래킹 데이터 전송
📌
발행 시 Segment별 속성 추가(Append)가 가능합니다. Segment 할당 정보를 DE 필드로 포함시켜 이메일 발송 시 AMPScript에서 활용할 수 있습니다.

Dimension Package

Dimension Package는 Audience Builder가 Subscriber 데이터와 상호작용하는 방식을 정의합니다. 원시 속성(raw attributes)을 Attribute Library에서 읽기 쉬운 정보로 변환합니다.

💡
Dimension Package는 Contact Builder Attribute Group에 연결된 DE의 필드를 Audience Builder에서 사용할 수 있도록 매핑합니다. Attribute Group 설정이 선행되어야 Dimension Package가 정상 동작합니다.

Audience Builder 리포팅

Audience Builder는 표준 이메일 리포트 섹션에 Segment명 기준 캠페인 지표를 제공합니다. 어느 Segment가 높은 오픈율/클릭율을 보이는지 분석해 다음 캠페인 대상 선정에 활용하세요.

📌
성능 특성: 모든 Audience Builder 구현은 고유한 성능 특성을 갖습니다. 계정의 Contact 수, Attribute 수, 조건 복잡도에 따라 발행 시간이 달라집니다. 구현 전 팀과 성능 기대치를 사전에 논의하세요.

Contact Builder 개요

Contact Builder는 MCE 계정 내 모든 채널(Email, SMS, Push, Journey)에 걸친 Contact 데이터를 중앙에서 관리하는 앱입니다. DE 간 관계를 시각적으로 설계하고, Journey Builder의 Decision Split 조건 등에 필요한 Attribute Group을 구성합니다.

구성 요소기능
All Contacts계정 내 전체 Contact 뷰. 채널별·소스별·수정일별 필터 제공
Attribute GroupsDE 간 관계를 정의하는 데이터 모델. Journey Builder 조건 참조에 필수
Data DesignerAttribute Group을 시각적으로 구성하는 캔버스 툴
Data SourcesSystem(자동), Synchronized(CRM), Custom(외부) 데이터 소스 관리
Contact DeleteContact 레코드 전체 삭제(GDPR/개인정보보호법 대응)
⚠️
Contact Builder 사용자 권한: Access / View All Contacts / Delete Contacts / Administer Contact Data Model 4가지 권한이 별도로 관리됩니다. 구축 담당자는 Administer 권한, 일반 운영자는 Access + View만 부여하는 방식을 권장합니다.

Attribute Group 생성

Attribute Group은 Contact 레코드와 관련 DE를 연결하는 데이터 모델입니다. Journey Builder Decision Split에서 "회원 등급", "최근 구매일" 같은 DE 필드를 조건으로 사용하려면 반드시 해당 DE가 Attribute Group에 연결되어 있어야 합니다.

1
Contact Builder → Create Attribute Group 클릭
2
Attribute Group 이름 입력 후 저장
3
캔버스에서 Contact 레코드 아이콘 클릭 → Link DE로 첫 번째 DE 연결
4
관계(Cardinality) 선택: One-to-One / Many-to-One / Many-to-Many
5
연결 속성(Link Attribute) 선택: Contact 레코드와 DE의 같은 데이터 타입 필드 매핑 (필드명은 달라도 됨)
6
추가 DE 연결: 첫 번째 DE에 연결된 다른 DE를 체인 방식으로 추가

관계 설정 (Cardinality)

관계의미사용 예시
One-to-OneContact 1명당 DE 레코드 1개회원정보 DE, 등급 DE
One-to-Many (Many-to-One)Contact 1명당 DE 레코드 여러 개구매이력 DE, 이벤트 참여 DE
Many-to-Many복잡한 N:M 관계상품-카테고리 매핑 등 복잡한 연동
🚫
ContactID Mapping 주의: MCE 내부적으로 Contact를 식별하는 ContactID는 직접 MCE에서 추출한 데이터와만 연결할 계획이 아니라면 비활성화 상태로 두세요. 외부 데이터 소스와 연결할 때는 ContactKey(Subscriber Key)를 사용합니다.

Data Sources

Contact Builder의 Data Sources는 데이터가 어디서 왔는지 출처를 관리합니다. 세 가지 유형이 있습니다.

유형설명관리 방법
System MCE가 자동 생성하는 기본 데이터 구조 (Email Studio, Mobile 앱 등) 자동 관리. 수동 수정 불가
Synchronized Marketing Cloud Connect를 통해 Salesforce CRM에서 자동 동기화 동기화 대상 오브젝트·필드 선택, 주기 설정
Custom 외부 시스템 데이터를 식별하기 위해 수동으로 지정하는 출처 Attribute를 Custom 소스에 직접 할당

Synchronized Data Sources (CRM 연동)

Marketing Cloud Connect가 설치된 경우, Salesforce CRM의 Contact, Lead, Account, 커스텀 오브젝트 데이터를 MCE DE로 자동 동기화합니다.

1
Contact Builder → Data Sources → Synchronized 탭
2
동기화할 오브젝트 선택 (Contact, Lead, Account 우선 동기화 권장)
3
동기화할 필드 체크 → Next
4
레코드 범위 선택: 전체 / 이메일 있는 레코드만 / 특정 날짜 이후 생성 / Boolean 필터
5
저장 후 동기화 완료 대기 (처음에는 시간 소요)
⚠️
Synchronized DE 사용 제한: Synchronized Data Source로 생성된 DE는 Primary Sendable DE로 직접 사용할 수 없습니다. 발송 대상 DE는 별도 Standard DE를 만들고, Synchronized DE에서 Query Activity로 데이터를 복사하는 방식을 사용하세요.
📌
멀티 BU 환경: 동일 Salesforce System User로 연결된 모든 BU가 같은 동기화 데이터를 공유합니다. BU 5개를 초과하면 서버 성능 저하가 발생할 수 있습니다. 필드 추가·삭제 등 소스 변경은 모든 연결된 BU에 영향을 미칩니다.

All Contacts

All Contacts 탭은 MCE 계정 내 모든 Contact를 단일 뷰로 보여줍니다. Channel, Source, Type, Population 기준으로 필터링할 수 있습니다.

필터설명
All Contacts계정 내 전체 Contact
PopulationsPopulation에 할당된 Contact
Last Modified지정 기간 내 정보 변경된 Contact
Source특정 방식으로 유입된 Contact (Email, API, Import 등)
Channel특정 채널과 연결된 Contact (Email Studio, MobileConnect 등)
TypeContact 유입 소스 유형. 기본값 (0)은 사용자 입력

Contact Delete

GDPR, 개인정보보호법 등의 삭제 요청에 대응합니다. Contact Delete는 계정 내 해당 Contact의 모든 데이터를 영구 삭제합니다 — Sendable DE 레코드, 트래킹 데이터, 선호 설정까지 포함됩니다.

🚫
복구 불가: 삭제된 Contact 정보는 복원할 수 없습니다. 삭제 전 반드시 해당 DE를 백업 파일로 내보내세요. 삭제 후 동일 ContactKey가 다시 추가되면 새로운 고유 ID를 받으며, 이전 데이터는 연결되지 않습니다.

Contact Delete 절차

1
Contact Builder → 설정에서 Contact Delete 활성화 (Admin 권한 필요)
2
삭제 대상 Contact 확인 및 백업 DE 내보내기
3
Contact Builder → All Contacts에서 해당 Contact 선택 후 Delete 실행
4
삭제 진행 상태 확인: Contact Builder → Contact Analytics (최근 30일 삭제 이력 조회)
5
삭제 요청 처리 이력 보관 DE에 일시·요청 경로 기록 (법적 증빙)

Best Practices

  • 1회 최대 100만 건 삭제 가능. 초과 시 여러 번 나누어 실행
  • 불필요한 Sendable DE 사전 정리 → 삭제 속도 향상
  • Enterprise 2.0 계정: 삭제는 반드시 Parent BU에서 시작
  • 삭제 전 진행 중인 Journey에서 해당 Contact 처리 방침 확인 (자동 제거됨)
  • Suppression Period(억제 기간) 동안은 해당 Contact 정보에 접근 불가하며, Final Deletion 후 완전 제거
📌
CRM Sync 중지와 Contact Delete 차이: Salesforce Connect에서 Contact 동기화를 중지하면 Synchronized DE에서는 제거되지만 Contacts 프레임워크에는 남아 있습니다. 완전 삭제가 필요하면 Contact Delete를 별도로 실행해야 합니다.
