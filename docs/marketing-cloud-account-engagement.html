Marketing Cloud Account Engagement (MCAE)

구 이름 Pardot. B2B 마케팅 자동화 플랫폼으로, 리드 스코어링·그레이딩·너처링·Salesforce CRM 양방향 동기화가 핵심입니다. MCE(이메일/SMS 대량 발송 중심)와 목적이 다릅니다.
출처: Salesforce 공식 문서 Marketing Cloud Account Engagement (2026.06.23)

MCAE 개요

MCAE는 복잡한 B2B 세일즈 사이클을 지원하는 마케팅 자동화 플랫폼입니다. 잠재 고객(Prospect)의 웹사이트 방문, 이메일 열람, 폼 제출 등 모든 디지털 행동을 추적하고 점수화해, 영업팀이 구매 가능성 높은 리드에 집중할 수 있도록 합니다.

MCE vs MCAE — 언제 어떤 걸 쓰나?

Marketing Cloud Engagement (MCE)
  • 대상: B2C, 대량 구독자
  • 채널: 이메일, SMS, Push, In-App, Ads
  • 핵심: 개인화 이메일 대량 발송, 실시간 캠페인
  • 리드 관리: 기본 Subscriber 중심
  • CRM 연동: Marketing Cloud Connect (선택)
  • 발송 단위: 수백만 건/일 가능
  • 개인화: AMPScript, Server-Side JS
Marketing Cloud Account Engagement (MCAE)
  • 대상: B2B, 소규모 고가치 리드
  • 채널: 이메일, Landing Page, Form
  • 핵심: 리드 점수화, 너처링, CRM 통합
  • 리드 관리: Prospect 생애주기 전체 추적
  • CRM 연동: 기본 내장 (필수), 양방향 실시간 동기화
  • 발송 단위: 수천~수만 건/캠페인
  • 개인화: HML(Handlebars-like), 동적 콘텐츠
📌
MCE + MCAE 동시 운영: 실제로 두 제품을 함께 쓰는 기업이 많습니다. MCAE로 초기 리드를 너처링해 Hot Lead로 발전시키고, 계약 후 온보딩·리텐션 이메일은 MCE로 대량 발송하는 패턴이 일반적입니다.

Prospects & Visitors

개념설명비고
Visitor 웹사이트를 방문했지만 아직 식별되지 않은 익명 사용자. IP 기반으로 회사명을 역방향 조회 쿠키 기반. 폼 제출 시 Prospect로 전환
Prospect 이메일 주소가 확인된 마케팅 데이터베이스의 개인. 모든 활동이 추적됨 Salesforce Contact 또는 Lead와 매핑
Prospect Activity 이메일 열람, 클릭, 폼 제출, 페이지 방문 등 모든 행동 이력 Scoring 계산의 원천 데이터
Do Not Email 발송 수신 거부 상태. 이 상태의 Prospect에는 마케팅 이메일 발송 불가 Operational 이메일은 발송 가능
Opted Out 수신 거부 처리된 Prospect CRM의 Opted Out 체크박스와 동기화
💡
Prospect 생성 방법: (1) MCAE 폼/랜딩페이지 제출, (2) 이메일 링크 클릭으로 쿠키 심기, (3) API를 통한 직접 생성, (4) CSV/임포트, (5) Salesforce Contact/Lead가 MCAE로 동기화될 때 자동 생성

Lists (정적·동적)

Lists는 이메일 발송 대상 또는 Engagement Studio 입장 조건에 사용됩니다.

유형설명용도
Static List 직접 수동 추가/제거하거나 임포트로 구성하는 리스트 특정 이벤트 참석자, 타겟 계정 리스트
Dynamic List 규칙 기반 자동 관리. 조건에 맞는 Prospect가 자동 추가·제거됨 리드 점수 50점 이상, 특정 회사명, 특정 직책
Suppression List 발송 제외 대상 리스트. Engagement Studio 수신 억제에 사용 경쟁사 이메일 도메인, 이미 고객인 계정
Test List 실제 발송 전 테스트 전송 전용 리스트 마케팅 팀 내부 테스트
Seed List 모든 이메일 발송 시 자동으로 포함되는 수신자 리스트 감사·컴플라이언스 목적 내부 CC
📌
Dynamic List 활용 팁: 리드 점수·그레이드 조건으로 Dynamic List를 만들면 영업 적격 리드(SQL)를 자동 분류할 수 있습니다. 예: Score ≥ 100 AND Grade ≥ B → "Hot Lead" Dynamic List → CRM에 자동 Task 생성

Scoring Categories

제품 라인, 관심 영역별로 별도 점수 카테고리를 운영할 수 있습니다. 예를 들어 "클라우드 제품" 폼 제출은 클라우드 카테고리 +50점, "교육" 페이지 방문은 교육 카테고리 +1점 식으로 분리 추적합니다.

Scoring Category는 Prospect 상세 화면에 별도 집계되어, 영업 담당자가 어느 제품에 관심이 있는지 한눈에 파악할 수 있습니다.

Grading — 프로필 적합도

Grading은 행동(Activity)이 아닌 프로필 정보(직책, 업종, 회사 규모 등)가 이상적인 고객 프로필(ICP)과 얼마나 일치하는지 나타냅니다.

등급의미대응 방식
A+완벽한 프로필 적합즉시 영업 배정
A높은 적합도우선 순위 너처링 → 영업 패스
B양호한 적합도중간 강도 너처링
C보통 적합도일반 너처링 유지
D낮은 적합도관계 유지 자동화
F부적합 프로필마케팅 비용 최소화

Profile & Criteria 설정

Admin → Profiles에서 이상적 고객 프로필을 정의합니다. 각 Profile에는 Criteria(조건)와 점수 가중치를 설정합니다.

  • 직책(Job Title): CTO, VP Engineering → Grade +, 인턴 → Grade -
  • 업종(Industry): 금융·제조·의료 → Grade +, 개인 사용자 → Grade -
  • 회사 규모(Employees): 1,000명 이상 → Grade +
  • Country: 한국·일본·미국 → Grade +, 특정 지역 → Grade -
💡
MQL 정의 예시: Score ≥ 100 AND Grade ≥ B → MQL(Marketing Qualified Lead). 이 조건을 Dynamic List로 만들고 Engagement Studio에서 Salesforce Task 자동 생성 → 영업팀 배정 플로우를 구성하세요.

Engagement Studio

Engagement Studio는 MCAE의 핵심 너처링 도구입니다. 시간 지연, 이메일 발송, CRM 액션, 리드 분기 등을 시각적 플로우로 구성해 완전 자동화된 너처링 프로그램을 운영합니다.

📌
Engagement Studio ≠ Journey Builder: 목적이 다릅니다. Engagement Studio는 B2B 리드 너처링(수주 사이클 전반)에 특화되어 있으며, 기본 단위가 "며칠 후 다음 액션"입니다. Journey Builder는 B2C 실시간 개인화 여정에 최적화되어 있습니다.

프로그램 설정 주요 옵션

  • Recipient Lists: 프로그램 대상 리스트 (Static 또는 Dynamic)
  • Suppression Lists: 발송 제외 대상 리스트
  • Schedule: 발송 허용 요일·시간 설정 (영업일 오전 9시~오후 6시 권장)
  • Allow Re-entry: 동일 Prospect의 프로그램 재참가 허용 여부
  • Pause Program: 전체 프로그램 일시 정지 가능

Forms & Landing Pages

MCAE Forms는 웹사이트에 직접 임베드하거나 MCAE Landing Page에 삽입해 Prospect 정보를 수집합니다. 제출 즉시 Prospect 레코드가 생성·업데이트되고 Engagement Studio 트리거가 발동합니다.

Form 주요 기능

기능설명
Progressive Profiling 재방문자에게 이미 아는 필드는 숨기고 새 필드 표시. 매 방문마다 추가 정보 수집
Dependent Fields 다른 필드 값에 따라 조건부로 표시되는 필드. 예: Industry = Healthcare → 병원 규모 필드 표시
Pre-fill 이미 알고 있는 Prospect 정보 자동 채우기. 마찰 감소, 전환율 향상
Honeypot 사람 눈에 보이지 않는 필드. 봇이 채우면 스팸으로 처리
Always Display Values 체크 시 기존 값과 상관없이 항상 새 값으로 덮어쓰기
Completion Actions 제출 후 즉시 실행되는 자동화: Score 변경, List 추가, CRM 알림, 태그 추가 등

Form Handlers

외부 웹사이트에 이미 자체 폼이 있다면 MCAE Form을 대체할 필요 없이 Form Handler를 사용합니다. 외부 폼 제출 데이터를 MCAE에 POST로 전달해 Prospect 레코드를 생성·업데이트합니다.

1
MCAE → Marketing → Forms → Form Handlers → Add Form Handler
2
외부 폼의 필드명과 MCAE Prospect 필드 매핑
3
Completion Actions 설정 (이메일, 담당자 알림, CRM Task 생성)
4
Endpoint URL 복사 → 외부 폼 action 속성에 붙여넣기
5
허용 도메인(Success/Error Location) 설정 → 스팸 방지
⚠️
kiosk mode: 같은 디바이스에서 여러 사람이 제출할 수 있는 환경(이벤트 등록 키오스크 등)에서는 Kiosk Mode를 활성화하세요. 쿠키 없이 각 제출을 독립 레코드로 처리합니다.

유형설명특이사항

Engagement Studio Email Engagement Studio 플로우 내 Action으로 발송 플로우 단계별 발송. 개인 발송이므로 일정 달라짐
Operational Email 구매 확인, 계정 알림 등 거래적 성격의 이메일 Opted Out 상태에도 발송 가능. 오남용 주의
A/B Test 2개 버전 이메일의 오픈율·클릭율 비교 테스트 샘플 비율 설정 후 승자 자동 발송 또는 수동 선택
Scheduled Send 특정 날짜/시간에 자동 발송 예약 발송 전 취소 가능. 시간대는 계정 기본 시간대 기준
📌
Einstein Send Time Optimization (STO): 각 Prospect의 과거 이메일 오픈 패턴을 분석해 개인별 최적 발송 시간을 자동 계산합니다. B2B에서 오픈율을 평균 15~20% 향상시키는 효과가 있습니다.

Salesforce CRM 동기화

MCAE는 Salesforce CRM과 기본 양방향 동기화가 이루어집니다. Prospect ↔ Contact/Lead 가 자동으로 연결되어, 마케팅 활동 데이터가 실시간으로 CRM에 반영됩니다.

방향동기화 내용
MCAE → CRM Prospect 속성 변경(이름, 회사, 직책 등), Score, Grade, 활동 이력, Opt-out 상태
CRM → MCAE Contact/Lead 필드 업데이트, Owner 변경, Campaign Membership, 사용자 정보
충돌(Conflict) 기본: CRM 값 우선. Prospect Last Modified 기준으로 덮어쓰기. 필드별 재정의 가능

User Sync

Salesforce User를 MCAE User로 자동 동기화합니다. Prospect의 Owner가 변경되면 영업 담당자에게 알림이 발송됩니다.

⚠️
User Sync 활성화 주의: User Sync를 활성화하면 Salesforce에 존재하지 않는 MCAE 전용 사용자는 비활성화됩니다. 활성화 전 사용자 목록을 반드시 검토하세요. 또한 사용자 역할(Role) 구조가 MCAE에도 반영됩니다.

Connected Campaigns

Connected Campaigns는 MCAE 캠페인과 Salesforce Campaign을 1:1 연결합니다. 활성화하면 MCAE의 모든 캠페인 멤버십이 Salesforce Campaign에 실시간 반영되어, Salesforce Opportunity Influence와 Attribution 보고서를 사용할 수 있습니다.

1
MCAE Admin → Connectors → Salesforce → Connected Campaigns 활성화
2
기존 MCAE 캠페인을 Salesforce Campaign과 수동 매핑
3
신규 캠페인은 Salesforce에서 생성 → MCAE에 자동 연결
📌
First Touch / Last Touch Attribution: Connected Campaigns가 활성화되어야 MCAE B2B MA Attribution 보고서에서 캠페인별 Opportunity 기여도를 정확히 측정할 수 있습니다.

Einstein 기능

MCAE Einstein은 머신러닝 기반 AI 기능을 추가 제공합니다. Growth, Plus, Advanced, Premium 에디션에서 사용 가능합니다.

기능설명활용
Einstein Behavior Scoring 최근 활동 패턴을 AI가 분석해 구매 의도 점수(0~100) 실시간 산출 기존 Scoring이 누적 합산이라면, Behavior Score는 최근 행동 패턴 중심. 두 지표를 함께 활용
Einstein Lead Scoring Salesforce의 과거 승인된 Opportunity 패턴을 학습해 유사 리드 자동 식별 영업팀에 "이 리드와 유사한 패턴의 리드가 과거 xx%로 성사됨" 인사이트 제공
Send Time Optimization (STO) 개인별 최적 이메일 발송 시간 자동 결정 List Email 설정 시 STO 옵션 활성화. 오픈율 향상
Engagement Frequency Prospect의 이메일 피로도 자동 감지. 과잉 발송 경고 주 5회 이상 발송 시 수신거부율 급증 예방
Campaign Insights 캠페인 결과에 영향을 미친 상위 요인 자동 분석 "이번 캠페인에서 오픈율이 높았던 직책은 CTO(35%)" 형태의 자동 분석 제공
Attribution AI Opportunity 영향에 기여한 캠페인을 AI가 가중치 산정 First Touch / Last Touch / Linear 외 AI 방식으로 멀티터치 Attribution 분석 가능
✨
Einstein 사용 전 필수 조건: 충분한 데이터 볼륨이 필요합니다. Behavior Scoring은 최소 수천 개의 Prospect 활동 이력, Lead Scoring은 100건 이상의 Closed Won Opportunity가 있어야 의미 있는 모델이 생성됩니다. 초기 구축 단계에서는 기본 Scoring/Grading을 먼저 안정화하세요.

B2B Marketing Analytics (B2BMA)

B2BMA는 MCAE + CRM 데이터를 통합하는 Salesforce Analytics 기반 리포팅 앱입니다. Advanced 이상 에디션에서 제공됩니다.

  • Pipeline Report: 캠페인별 생성된 Opportunity 파이프라인 금액
  • Attribution Report: 멀티터치 기여도 분석 (First Touch, Last Touch, Even Distribution, Position-Based)
  • Engagement Studio Report: 프로그램 단계별 이탈·전환 퍼널
  • Prospect Lifecycle Report: 리드에서 MQL → SQL → Opportunity 전환율 추적
  • Account Dashboard: 계정(Account) 단위 참여도 집계
📌
Connected Campaigns 선행 필수: B2BMA에서 의미 있는 Attribution 분석을 하려면 Connected Campaigns가 활성화되어 있어야 합니다. Opportunity Influence 설정(Salesforce CRM → 캠페인 영향 추적 활성화)도 함께 확인하세요.

Engagement History Dashboard

MCAE 공식 Salesforce CRM 대시보드입니다. Contact/Lead 레코드 페이지에 직접 삽입되어, 영업 담당자가 CRM을 떠나지 않고 마케팅 활동 이력을 실시간으로 확인할 수 있습니다.

  • 최근 이메일 수신·오픈·클릭 이력
  • 방문한 랜딩페이지 목록
  • 제출한 폼 내역
  • 참가한 캠페인 목록
  • 현재 Score와 Grade
💡
영업 담당자 활용: 영업 콜 전에 Engagement History Dashboard로 리드가 어떤 이메일을 열람했는지, 어떤 페이지를 방문했는지 확인하면 맞춤형 대화가 가능합니다. "저희 클라우드 솔루션 페이지를 3번 방문하셨던데요…" 같은 맥락 있는 영업이 가능해집니다.
