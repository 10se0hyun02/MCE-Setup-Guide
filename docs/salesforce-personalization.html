Salesforce Personalization

최종 업데이트: 2026.06.19 · 출처: Salesforce 공식 문서 (224페이지)

🎯 개요

Salesforce Personalization은 Data 360(Data Cloud) 위에서 작동하는 Customer 360 앱입니다. 머신러닝 기반 추천과 규칙 기반 개인화를 결합하여, 웹·이메일·모바일·Sales Cloud 등 전 채널에서 고객별 맞춤 콘텐츠와 오퍼를 실시간으로 제공합니다.

ℹ️
전 채널 개인화 엔진기존 Marketing Cloud Personalization(구 Interaction Studio)과는 별개 제품으로, Data 360을 기반으로 처음부터 새로 설계된 네이티브 앱입니다. 기존 제품에서 마이그레이션하려면 별도 절차가 필요합니다.

지원 채널

  • 웹 — Interactions SDK(JavaScript)를 통한 실시간 웹 페이지 개인화
  • 이메일 캠페인 — Marketing Cloud Engagement 연동, 오픈 시점 콘텐츠 결정
  • 모바일 앱 — SDK 연동 후 인앱 개인화
  • Sales / Service Cloud — Einstein Next Best Action, Personalization Card
  • 서드파티 채널 — Decisioning API를 통한 외부 시스템 연동

📖 핵심 용어

Calculated Insight

Data 360 세그먼트·집계 데이터를 기반으로 계산된 인사이트 값. 추천과 규칙에서 조건 변수로 사용.

Data Graph

Identity Resolution으로 통합된 고객 단일 프로필. 행동 이력·선호도·속성을 모두 포함.

Decisioning API

외부 앱·채널에서 추천 결과를 요청할 수 있는 REST API. 서드파티 CMS, 키오스크 등에 활용.

Personalization Point

개인화 결과를 표시할 채널 내 위치. 웹 페이지 영역, 이메일 슬롯, API 엔드포인트 등.

Recommender

ML 알고리즘으로 상품/콘텐츠 추천을 생성하는 엔진. 최대 10개까지 생성 가능.

Targeting Rule

개인화 노출 조건을 정의하는 규칙. 세그먼트, Calculated Insight, URL 조건 등 복합 설정.

Sitemap

웹사이트 구조와 트래킹 대상을 정의하는 설정 파일. Data 360 SDK와 연동됩니다.

Content Schema

개인화에 사용할 콘텐츠 항목(상품·기사·오퍼)의 데이터 구조 정의.

Experience

특정 Personalization Point에서 보여줄 콘텐츠 변형(Variation) 묶음.

Calculated Affinity

고객이 특정 카테고리/태그에 보인 관심도를 수치화한 값. 추천 필터링에 사용.

Engagement Signal

클릭·구매·조회 등 Data 360에서 수집된 고객 행동 이벤트.

Einstein Studio

커스텀 ML 모델 예측값을 개인화 결정에 통합하는 연동 레이어.

⚡ 시스템 제한

  • 10개Recommender (org당 최대)
  • 60,000/분추천 요청 처리 한도
  • 24개1회 API 응답의 최대 추천 항목 수
  • 10개필터 조건 (Recommender당 최대)
  • 50개Targeting Rule (org당 최대)
  • 100개Personalization Point (org당 최대)
  • 20개Content Schema (org당 최대)
  • 5MBSitemap 파일 최대 크기
⚠️
Recommender 한도 주의org당 10개의 제한은 늘리기 어렵습니다. 비슷한 로직을 가진 추천은 Targeting Rule로 분기하는 방식으로 설계해 한도를 절약하세요.

🔧 Data 360 연동 설정

Salesforce Personalization은 반드시 Data 360(Data Cloud) 활성화 후 설정해야 합니다.

  1. Data 360 활성화 — Setup → Data Cloud Setup에서 기본 설정 완료 및 Identity Resolution 구성
  2. CRM 커넥터 생성 — Salesforce CRM Connector 추가, Contact·Lead·Account 동기화 설정
  3. Personalization 데이터 키트 설치 — AppExchange에서 Salesforce Personalization Data Kit 설치
  4. 데이터 스페이스 선택 — Personalization Setup에서 사용할 Data Space 지정
  5. 기본 이메일 채널 설정 — Salesforce 지원을 통해 Default Email Channel 활성화 요청
  6. Interactions SDK 설치 — 웹사이트에 JavaScript SDK 태그 삽입, Sitemap 설정
  7. Identity Resolution 매핑 — 이메일·전화·쿠키 기준 통합 설정 검토 및 보완
ℹ️
설정 소요 시간기본 활성화는 최대 15분이지만, Identity Resolution이 첫 번째 배치 작업을 완료하려면 데이터 볼륨에 따라 수 시간이 걸릴 수 있습니다.

Calculated Affinity

수집된 시그널을 바탕으로 고객이 특정 카테고리·태그에 보인 관심도를 0~100 사이의 수치로 계산합니다. 이 값은 Recommender 필터나 Targeting Rule 조건에서 직접 참조할 수 있습니다.

🤖 추천 엔진 (Recommender)

ML 기반 추천 엔진으로 상품·콘텐츠·오퍼를 자동으로 랭킹하여 제공합니다.

추천 알고리즘 유형

알고리즘설명적합 사례
Collaborative Filtering유사 고객이 구매/조회한 항목 추천상품 추천, 콘텐츠 추천
Content-Based고객이 좋아한 항목의 속성과 유사한 항목 추천카탈로그 다양성 확보
Popularity-Based전체 고객 기준 인기 순위 추천신규 방문자, 콜드스타트
Context-Aware현재 페이지 컨텍스트(조회 중인 상품) 기반 추천상세 페이지 크로스셀
Rules-Based BoostML 결과에 비즈니스 규칙(재고·마진 등) 가중치 부여재고 소진 우선 노출

필터 조건 설정

  • 카탈로그 속성 기반 필터 (카테고리, 가격대, 태그)
  • 고객 Affinity 기반 필터 (관심도 임계값 이상)
  • 구매 이력 제외 (이미 구매한 상품 제외)
  • 재고 상태 필터 (품절 상품 자동 제외)
  • 비즈니스 규칙 (마진율, 프로모션 여부)
💡
필터 조건 10개 제한 대응 방법자주 사용하는 조건 조합은 Data 360 Calculated Insight로 미리 집계해 하나의 변수로 만들면, 복수 조건을 단일 조건으로 압축할 수 있습니다.

개인화 결과가 실제로 표시될 위치를 정의합니다. 채널·화면·위치별로 구성합니다.

채널Point 유형설명

ℹ️

📣 캠페인 & 실험

캠페인 구성 요소

  • Targeting Rule — 어떤 고객에게 보여줄지 조건 정의
  • Experience — 조건에 맞는 고객에게 어떤 콘텐츠를 보여줄지 변형(Variation) 설정
  • Schedule — 캠페인 시작·종료 일시, 우선순위
  • Goal — 성과 측정 기준 (클릭율, 전환율, 수익)

A/B 테스트 (Experimentation)

하나의 Personalization Point에 복수의 Experience를 설정하고 트래픽을 분할하여 성과를 비교합니다. 통계적 유의성에 도달하면 Salesforce가 자동으로 승리 변형을 선택하거나, 수동으로 배포를 결정할 수 있습니다.

⚠️
Personalization Card 라이선스는 Experimentation 미지원A/B 테스트 기능은 Full Personalization License에서만 사용 가능합니다. Personalization Card 고객은 단일 Experience만 실행할 수 있습니다.

🤖 Agentforce 통합

Salesforce Personalization의 추천 결과와 고객 인사이트를 Agentforce 에이전트에서 활용할 수 있습니다.

통합 시나리오

  • Service Cloud 에이전트 — 상담 중 Personalization 추천 상품/오퍼를 에이전트 화면에 제공
  • Sales Cloud 에이전트 — Personalization Card를 통해 Opportunity에 맞는 Next Best Offer 제안
  • Marketing 에이전트 — 세그먼트 인사이트 기반으로 캠페인 콘텐츠 자동 생성
  • Autonomous Agent — Personalization 데이터를 참고하여 자율적으로 후속 액션 결정

설정 방법

  1. Agentforce Setup에서 Personalization Action 활성화
  2. Agent Topic에 Personalization 관련 Action 추가
  3. Personalization Point를 Agentforce 채널로 구성
  4. 데이터 권한(Object Permission) 검토 — Agentforce 시스템 사용자에게 Personalization 객체 읽기 권한 부여

⚙️ 배치 개인화 (Batch Personalization)

실시간 API 호출 없이 세그먼트 단위로 사전 생성된 추천 결과를 대량 적용하는 방식입니다. 이메일 캠페인처럼 발송 시점에 미리 추천을 계산해야 할 때 사용합니다.

배치 방식 vs 실시간 방식 비교

항목배치 개인화실시간 개인화
결정 시점발송 전 사전 계산고객이 화면을 로드하는 순간
크레딧 소비세그먼트 단위 (상대적으로 적음)노출당 1 크레딧
데이터 최신성계산 시점 기준 (다소 지연)실시간 최신 데이터 반영
적합 채널이메일, 푸시 알림, SMS웹, 앱, 실시간 채팅
💡
이메일에는 배치 방식 권장MCE 이메일 발송과 Personalization을 연동할 때는 배치 방식이 안정적입니다. 실시간 오픈 시점 개인화(Open-Time Personalization)를 원하는 경우에만 실시간 방식을 사용하고, 크레딧 소비량을 사전에 계획하세요.

📊 분석 & 보고

기본 제공 지표

  • 노출 수(Impressions) — Personalization Point가 실제로 표시된 횟수
  • 클릭율(CTR) — 추천 항목 클릭 비율
  • 전환율(Conversion Rate) — 추천 후 목표 행동 완료 비율
  • Revenue per Impression — 노출당 창출 수익 (이커머스)
  • Lift — 비개인화 대비 개인화 성과 향상률
  • Recommender Hit Rate — 추천 요청 중 결과가 반환된 비율

Einstein Studio 예측 통합

Data Cloud의 Einstein Studio에서 생성한 ML 예측 모델(이탈 확률, CLV 등)을 Personalization 결정 로직에 통합할 수 있습니다. 예를 들어 이탈 위험 고객에게는 할인 오퍼, 고가치 고객에게는 프리미엄 상품을 우선 추천하는 규칙을 구성할 수 있습니다.
