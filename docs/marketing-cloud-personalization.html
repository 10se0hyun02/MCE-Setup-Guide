Marketing Cloud Personalization

구 Interaction Studio · 최종 업데이트: 2026.06.19 · 출처: Salesforce 공식 문서 (667페이지)

🎭 개요

Marketing Cloud Personalization(MCP, 구 Interaction Studio)은 고객 경험을 실시간으로 시각화·추적·관리하는 개인화 플랫폼입니다. 적절한 순간에 고객이 선호하는 방식으로 가치 있는 참여를 유도하며, MCE의 개인화 역량을 확장합니다.

⚠️
신규 도입 고객은 Salesforce Personalization 권장MCP(구 Interaction Studio)는 레거시 아키텍처(그래프 기반) 제품입니다. 신규 프로젝트에는 Data 360 기반의 Salesforce Personalization 도입을 권장합니다. 기존 MCP 고객은 MCN for Personalization을 통해 Data 360으로 점진적 전환이 가능합니다.

주요 역량

  • 실시간 고객 프로필 구성 및 행동 추적
  • 웹·이메일·모바일·서버사이드 채널 개인화
  • Einstein Recipes·Einstein Decisions 기반 AI 추천
  • A/B·MVT 테스트 내장
  • 카탈로그 관리 (상품·콘텐츠·오퍼)
  • 세그먼트 기반 캠페인 타겟팅

👤 통합 프로필 & ID 관리

MCP는 웹·이메일·앱 등 다양한 채널에서 수집한 데이터를 단일 고객 프로필로 통합합니다.

ID 해석 (Identity Resolution)

  • 익명 방문자 — 쿠키 ID로 추적, 로그인 시 인증된 ID와 매칭
  • 인증된 방문자 — 이메일·전화·CRM ID 기반 통합
  • Cross-Device 매핑 — 동일 고객의 PC·모바일·앱 행동 연결

프로필 구성 요소

구성 요소내용
기본 속성이름, 이메일, 전화, 생년월일 등 인구통계 정보
행동 이력페이지 조회, 상품 클릭, 구매, 폼 제출 이력
카탈로그 관심도특정 카테고리·태그에 대한 친밀도(Affinity) 점수
세그먼트 멤버십소속 세그먼트 목록 (실시간 갱신)
커스텀 프로필 객체비즈니스 맞춤 추가 속성 정의

📦 카탈로그 & 프로모션

카탈로그 (Catalog)

추천·개인화의 대상이 되는 콘텐츠 항목(상품·기사·오퍼 등)의 데이터 구조를 정의합니다.

  • 상품(Product) — ID, 이름, 가격, 카테고리, 재고 상태, 이미지 URL
  • 기사(Article) — ID, 제목, 태그, 발행일, 저자
  • 오퍼(Offer) — ID, 할인율, 유효기간, 자격 조건
  • 커스텀 카탈로그 객체 — 비즈니스 맞춤 항목 정의 (예: 강의, 부동산 매물)

카탈로그 데이터 업데이트 방법

  • ETL (파일 업로드) — CSV/JSON 파일로 배치 업데이트
  • Web SDK — 페이지 로드 시 실시간 카탈로그 항목 업데이트
  • API — REST API로 개별 항목 즉시 업데이트
  • Amazon S3 커넥터 — Data 360으로 전환 시 S3 기반 자동 업데이트

사용자 프로필 객체 (User Profile Object)

고객의 카탈로그 관련 행동을 기록하는 구조. 예: "고객 A는 '스포츠' 카테고리를 7회 조회했다"는 정보가 사용자 프로필 객체에 기록되어 추천 엔진의 입력값이 됩니다.

프로모션 (Promotion)

특정 고객 세그먼트에게 특별 혜택을 제공하는 프로모션을 정의하고 개인화 캠페인에 연결할 수 있습니다.

🎯 세그먼트 정의

MCP에서 세그먼트는 캠페인 대상 조건을 정의합니다. 다양한 기준을 조합하여 세밀하게 타겟팅할 수 있습니다.

🧠 Einstein AI 개인화

MCP에는 두 가지 핵심 AI 엔진이 내장되어 있습니다: Einstein Recipes(추천)와 Einstein Decisions(의사결정).

🍳 Einstein Recipes

ML 기반 상품·콘텐츠 추천을 생성하는 엔진입니다. '재료(Ingredients)'를 조합해 추천 로직을 직관적으로 구성합니다.

Ingredient 유형

Ingredient설명
Viewed Together현재 보고 있는 상품과 함께 조회된 상품
Purchased Together함께 구매된 상품 (크로스셀)
Co-Browse유사 방문 패턴을 가진 사용자들이 본 상품
Trending최근 급상승 조회 상품
Top Sellers판매량 상위 상품
Recently Viewed해당 고객이 최근에 조회한 상품
Related Catalog Objects동일 카테고리·태그의 관련 항목

Exclusion (제외 규칙)

  • 이미 구매한 상품 제외
  • 품절 상품 자동 제외
  • 특정 카테고리 항목 제외

Boost (가중치)

  • 고마진 상품 우선 노출
  • 프로모션 상품 가중치 부여
  • 신상품 우선 노출

⚡ Einstein Decisions

단순 추천을 넘어 여러 오퍼 중 고객에게 가장 적합한 단일 오퍼를 AI가 선택하는 의사결정 엔진입니다.

Einstein Decisions vs Einstein Recipes 차이

항목Einstein RecipesEinstein Decisions

입력 데이터행동 이력, 카탈로그 관계오퍼 성과, 고객 프로필, 비즈니스 제약
최적화 목표CTR, 전환율기대 수익, 고객 가치
적합 시나리오상품 추천 위젯, 관련 기사Next Best Offer, 프로모션 선택

📣 채널별 캠페인

웹 캠페인 (Web Campaign)

웹 페이지의 특정 영역(Zone)에 개인화 콘텐츠를 표시합니다. JavaScript SDK가 DOM을 실시간으로 조작합니다.

  • 인라인 교체 — 기본 콘텐츠를 개인화 콘텐츠로 교체
  • 팝업/배너 — 조건 충족 시 레이어 팝업 또는 상단 배너 표시
  • 오버레이 — 화면 전체를 덮는 전면 오퍼 표시

서버사이드 캠페인

API 호출로 개인화 결과를 수신하여 서버에서 렌더링합니다. 헤드리스 CMS, 모바일 앱, IoT 기기 등에 적합합니다.

오픈 시점 이메일 개인화 (Open-Time Personalization)

이메일을 열어보는 순간의 상황(시간, 위치, 날씨, 재고 상태)에 따라 표시되는 이미지와 콘텐츠가 실시간으로 결정됩니다.

💡
오픈 시점 개인화 활용 예시• 오전에 열면 "Good Morning" 이미지, 오후에 열면 "Good Afternoon" 이미지
• 이메일 작성 시점에 재고가 있었지만 오픈 시점에 품절인 상품은 대체 상품으로 자동 교체
• 고객의 현재 위치 기반 가장 가까운 오프라인 매장 정보 삽입

📊 리포트 & 분석

제공 리포트

  • 캠페인 성과 리포트 — 노출·클릭·전환·수익 등 핵심 지표
  • Lift 분석 — 개인화 적용 그룹 vs 비적용 그룹(Control) 비교
  • 세그먼트 분석 — 세그먼트별 행동 패턴 및 규모
  • 추천 성과 — Recipe별 CTR 및 전환율
  • Einstein Decisions 성과 — 의사결정별 오퍼 선택 비율 및 효과

A/B 테스트 (Experimentation)

캠페인 내에서 여러 변형을 동시에 실행하고 통계적 유의성에 근거하여 승리 변형을 결정합니다. Control 그룹(개인화 미적용)을 포함해 진짜 Lift를 측정할 수 있습니다.

🔄 MCN 전환 (Data 360으로 마이그레이션)

기존 MCP 데이터와 카탈로그를 Data 360(Data Cloud)으로 이전하여 Salesforce Personalization으로 전환하는 프로세스입니다.

ℹ️
병행 운영 가능MCP와 Salesforce Personalization은 SDK 동거(Coexistence) 설정을 통해 동일 웹사이트에서 병행 운영이 가능합니다. 빅뱅 전환 없이 점진적으로 이전할 수 있습니다.

전환 단계

  1. SDK 동거 활성화 — Personalization Setup에서 "Enable SDK Website Coexistence" 클릭
  2. Amazon S3 커넥터 생성 — Data 360에 P13N S3 Connection 커넥터 설정
  3. 데이터 모델 다운로드 — MCP 내 Data Model Download Tool로 .zip 생성
  4. Data 360 업로드 — Salesforce CLI로 .zip 파일을 Data 360에 업로드
  5. Data Kit 배포 — 두 개의 Data Kit(Profile & Engagement Kit, Catalog Kit) 순서대로 배포
  6. 카탈로그 데이터 이전 — Catalog Data Extract 도구로 기존 카탈로그 데이터 추출·업로드
  7. Sitemap 변환 — MCP Sitemap을 Data 360 호환 형식으로 수정
  8. 검증 및 파일럿 — 샌드박스에서 전체 개인화 플로우 검증

🗄️ Amazon S3 커넥터 설정

MCP 카탈로그 데이터를 Data 360으로 전송하기 위해 Amazon S3 커넥터를 사용합니다.
