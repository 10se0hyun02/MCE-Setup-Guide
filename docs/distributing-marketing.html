Distributed Marketing & Marketing Cloud Connect

최종 업데이트: 2026.06.19  |  번역: MCE 구축 가이드 팀

개요

Marketing Cloud Engagement의 마케팅 역량과 Salesforce CRM을 결합하여 통합된 고객 관계를 구축합니다.

제품핵심 역할주 사용 시나리오
Distributed Marketing 기업 마케팅 팀이 만든 콘텐츠를 파트너·딜러·어드바이저 등 분산 팀이 CRM에서 직접 개인화·발송 금융 어드바이저가 고객에게 맞춤 이메일 발송, 딜러 네트워크 프로모션
Marketing Cloud Connect MCE의 디지털 마케팅 역량과 Salesforce의 데이터 관리·세그먼트·캠페인 관리 도구 통합 Sales Cloud 내에서 MCE 이메일 발송, CRM 데이터 기반 Journey 자동화

Distributed Marketing

Distributed Marketing은 기업 마케팅과 파트너 네트워크 전반에 일관된 경험을 제공하기 위해 기업 마케팅, 직원, 채널 파트너, 브랜드 간의 격차를 해소합니다. 기업 마케터는 MCE에서 브랜드에 맞는 공동 작업 콘텐츠를 생성·공유하고, 어드바이저·파트너·딜러 등 분산 팀은 CRM(Sales, Service, Financial Services, Experience Cloud)에서 이 콘텐츠를 조회·개인화·발송합니다.

📌
Marketing Cloud Next for Engagement는 Distributed Marketing과 유사한 플랫폼 기반 기능을 포함합니다. 비마케팅 사용자에게 마케팅 승인 이메일 템플릿을 제공하려면 Distributed Marketing and Alerts를 참조하세요.

요구사항

필수 제품

  • MCE — Journey Builder 활성화 필수
  • Salesforce CRM — Sales/Service/Experience/Industry Cloud 중 하나 (Contact, Lead, Account, Campaign, Flow, SOAP API, Content 접근 권한 포함)
  • Salesforce Connect — 외부 데이터 소스 1개 연결 기능 (Distributed Marketing에 1개 라이선스 포함)

선택 제품

  • Marketing Cloud Connect — CRM에서 이메일 오픈 등 인게이지먼트 지표 확인, Journey Builder 고급 기능 활용
  • MobileConnect — SMS 발송 기능 추가
  • SSL — Private Domain 사용 고객이 Distributed Marketing Image Block 활용 시 필요 (HTTPS 전용)

Quick Send 설정

Quick Send는 사용자가 CRM에서 직접 단건 개인화 이메일 또는 SMS를 발송할 수 있는 기능입니다. 모든 Quick Send는 Journey Builder의 발송 엔진을 통해 처리됩니다.

Quick Send 발송 대상

  • 개별 Contact, Lead, Person Account (레코드·기회·Quick Send 메시지 레코드에서)
  • 리스트 뷰에서 최대 75명에게 동시 발송
  • Quick Send 메시지 레코드에서 최대 25,000명에게 동시 발송

Quick Send 활용 사례

  • 생일·명절 개인화 이카드
  • 자연재해 등 사건 후 공감 메시지
  • 고객 미팅 후 감사 메시지
  • 긴급 이슈 안내
  • 1:1 프로모션 또는 혜택 제공
  • 특정 주제 후속 이메일

Quick Send Journey 구성 요소

  • 이메일 또는 SMS 메시지 활동
  • 발신자 프로필 (이메일의 경우)
  • API 진입 소스용 데이터 익스텐션

Quick Send Journey 설정 단계

1
Distributed Marketing용 콘텐츠 생성 (MCE 환경)
  • 이메일 또는 SMS 메시지 생성 (필요 시 Distributed Marketing Content Blocks 또는 AMPscript 포함)
  • 콘텐츠 저장
2
진입 이벤트 데이터 익스텐션 생성 (MCE 환경)
  • Contact Builder와 Journey Builder 접근 권한 보유한 사용자가 생성
  • 발송자·수신자 정보를 저장하는 데이터 익스텐션
3
Quick Send용 Journey 생성 (MCE 환경)
  • Quick Send용으로 재진입(Re-entry) 허용 설정 필수
  • 단일 이메일 또는 SMS 활동 사용, 비메시지 활동·플로우 컨트롤 다수 추가 가능
4
Quick Send 워크플로우 설정 — Contact/Lead/Account 페이지 레이아웃에 Quick Send 컴포넌트 추가
5
콘텐츠 공유 — Distributed Marketing 관리에서 Journey 활성화 → 공유 규칙 설정

Campaign Send 설정

Campaign Send는 사용자가 Contact, Lead, Person Account를 단계별 Journey에 추가할 수 있습니다. 단순한 선형 드립 캠페인부터 참여 분기, AI, 복잡한 Journey까지 가능합니다.

Campaign Send 전략 고려사항

  • 하나의 Journey가 여러 캠페인을 지원할 수 있지만, 하나의 캠페인은 하나의 Journey만 연결 가능
  • 사용자별 고유 캠페인을 생성하고 Campaign Hierarchy로 관계화 권장
  • 사용자는 자신이 접근 권한을 가진 개인에게만 메시지 발송 가능

Campaign Send Journey 설정 단계

1
Distributed Marketing용 콘텐츠 생성 — 이메일(Content Builder) 또는 SMS(MobileConnect), CAN-SPAM 등 법적 요구사항 포함
2
진입 이벤트 데이터 익스텐션 생성
3
Campaign Send용 Journey 생성
  • API Entry Source 활동 추가 및 새 데이터 익스텐션으로 구성
  • Send Email 활동 추가 및 Distributed Marketing 발신자 프로필로 콘텐츠 구성
  • Contact Entry 설정
  • Journey 저장 및 활성화

Experience Cloud 설정

1
Campaign Messages 컴포넌트 추가 — Experience Builder에서 Campaign 오브젝트 페이지에 MCDM Campaign Messages 컴포넌트 추가
2
Pending Approvals 페이지 추가 — Linked Journey 오브젝트 페이지에 MCDM Pending Submissions 컴포넌트 추가
3
Settings 페이지 추가 — Standard Page 생성 후 MCDM User Settings 컴포넌트 추가 → 게시
4
User Settings 노출 — 탐색 메뉴에 Settings 메뉴 항목 추가
5
Quick Send 컴포넌트 추가 — 사용자가 단건 메시지 발송할 수 있도록 Distributed Marketing Quick Send 컴포넌트 추가

Marketing Cloud Connect

Marketing Cloud Connect는 MCE의 디지털 마케팅 역량과 Salesforce의 데이터 관리·세그먼트·캠페인 관리 도구를 결합합니다. Sales/Service Cloud에서 직접 MCE 도구를 사용하고, 일관된 브랜딩·메시지 경험을 보장하면서 더 많은 리드를 Salesforce로 유입시킵니다.

주요 기능

  • Salesforce CRM 데이터 기반 이메일 캠페인 생성·발송
  • Salesforce Lead·Contact·Campaign이 MCE 데이터 익스텐션과 동기화
  • Sales/Service Cloud에서 이메일 오픈·클릭 등 인게이지먼트 지표 확인
  • Salesforce에서 Journey Builder 활동 트리거
  • Sales Cloud, Service Cloud Activities(Journey Builder 활동) 사용 가능

설치 및 설정

1
MCE 계정 관리자 계정(전용 시스템 계정 권장) 생성
2
Salesforce CRM에 Marketing Cloud Connect 패키지 설치
3
MCE와 Salesforce CRM 간 인증 연결 설정
4
동기화할 Salesforce 오브젝트(Lead, Contact, Campaign 등) 매핑 설정
5
Send Classification 및 발신자 프로필 설정
6
테스트 이메일 발송으로 연동 확인
⚠️
Marketing Cloud Connect를 설정할 때 MCE 시스템 사용자는 전용 계정으로 설정해야 합니다. 직원 계정을 공유하면 계정 변경 시 연동이 중단될 수 있습니다.
