Referral Marketing

최종 업데이트: 2026.06.19 · 출처: Salesforce 공식 문서 (277페이지)

🎁 개요

Referral Marketing은 브랜드가 추천 프로그램(Referral Program)을 구축·운영할 수 있도록 지원하는 Salesforce 앱입니다. 기존 고객(Advocate)이 새 고객(Friend)을 소개하는 전형적인 MGM(Member Get Member) 구조를 상시 운영하거나 기간 한정으로 실행할 수 있습니다.

ℹ️
B2C 및 B2B 모두 지원일반 소비자 대상 추천 프로그램(B2C)뿐만 아니라, 기업 고객이 다른 기업을 추천하는 B2B Referral Management도 지원합니다. B2B의 경우 Sales Cloud와 연동됩니다.

주요 특징

  • 상시 운영(Evergreen) 및 기간 한정(Time-Bound) 프로그램 모두 지원
  • 추천 단계별 자동화 트리거 내장 (가입·구매·인증 등 각 단계에서 자동 보상 지급)
  • MCE와 연동하여 추천 링크 포함 이메일 자동 발송
  • 바우처(Voucher) 및 포인트 방식 보상 모두 구성 가능
  • OmniStudio 기반 셀프서비스 포털 구성 지원

📖 핵심 용어

Advocate (추천인)

추천 프로그램에 참여하여 친구·지인에게 브랜드를 소개하는 기존 고객.

Friend (피추천인)

Advocate의 추천 링크를 통해 브랜드에 처음 접속하는 신규 고객 후보.

Referral (추천 건)

Advocate와 Friend 사이에서 발생한 단일 추천 이벤트. 각 추천 건은 독립적으로 추적.

Voucher (바우처)

추천 보상으로 지급되는 할인 쿠폰·코드. 단일 사용 또는 복수 사용 설정 가능.

Promotion Template

추천 프로그램의 구조(단계·보상·기간)를 정의하는 템플릿.

Referral Promotion

Promotion Template을 기반으로 실제 실행 중인 추천 프로그램 인스턴스.

Transaction Journal

각 추천 이벤트(가입·구매·보상 지급)의 이력을 기록하는 감사 로그 객체.

Promotion Stage

추천 프로세스의 단계 (예: 친구 가입 → 친구 첫 구매 → 보상 지급).

Voucher Definition

바우처의 유형·할인율·유효기간을 정의하는 마스터 설정 객체.

🎫 보상 및 바우처

바우처 유형

  • 단일 사용 바우처 — 한 번만 사용 가능한 고유 코드. 남용 방지에 효과적.
  • 복수 사용 바우처 — 정해진 횟수만큼 재사용 가능한 코드.
  • 퍼센트 할인 — 결제 금액의 n% 할인.
  • 고정 금액 할인 — 특정 금액 차감.
  • 무료 배송 — 배송비 면제.
  • 포인트 적립 — Loyalty Program 포인트 연동.

바우처 발급 설정

  1. Voucher Definition 생성 — 할인 유형, 금액, 유효기간 설정
  2. Promotion Template에 보상 연결 — 각 단계(Stage)에 Voucher Definition 연결
  3. 발급 자동화 설정 — Flow Builder 또는 Apex Trigger를 통한 자동 발급 구성
  4. MCE 이메일 연동 — 바우처 코드를 이메일 본문에 동적으로 삽입
⚠️
바우처 만료일 설정 필수만료일 없는 바우처는 무기한 사용 가능하여 비용 리스크가 발생합니다. 반드시 합리적인 만료 기간을 설정하고, 만료 전 리마인더 이메일 발송 자동화도 구성하세요.

🔧 설치 및 설정

ℹ️
필수 사전 조건Referral Marketing 설치 전 반드시 OmniStudio 패키지가 org에 설치되어 있어야 합니다. 또한 MCE BU와의 연동이 필요합니다.
  1. OmniStudio 설치 확인 — Setup → Installed Packages에서 OmniStudio 존재 여부 확인
  2. Referral Marketing 패키지 설치 — AppExchange에서 Referral Marketing 검색 후 설치
  3. 권한 세트 할당 — 관리자·운영자·조회자별 Permission Set 할당
  4. MCE BU 연결 — Communication Method Admin이 MCE Business Unit 연동 설정
  5. 이메일 템플릿 구성 — 추천 링크·바우처 코드 포함 MCE 이메일 템플릿 작성
  6. Loyalty Program 생성 — Referral 타입으로 Loyalty Program 레코드 생성
  7. Promotion Template 설계 — 단계별 조건·보상 구조 정의
  8. 파일럿 테스트 — 샌드박스에서 전체 추천 흐름 검증 후 운영 배포

MCE 설정 포인트

  • API Event Entry Source — Salesforce Platform이 트리거 이벤트를 Journey Builder에 전달
  • AMPscript 추천 링크 삽입 — %%referralLink%% 형태의 Dynamic Content 변수 활용
  • 바우처 코드 삽입 — 보상 지급 시 MCE Transactional API 또는 Triggered Send로 바우처 코드 이메일 발송
  • UTM 파라미터 자동 부착 — 추천 링크에 Advocate ID를 파라미터로 포함하여 Attribution 추적
💡
Journey Builder vs Automation Studio추천 이메일은 고객 행동에 반응하는 이벤트 기반이므로 Journey Builder의 API Event Entry Source가 적합합니다. Automation Studio는 예약 발송 기반이라 추천 흐름과 맞지 않습니다.

🏢 B2B 추천 관리 (B2B Referral Management)

B2B 환경에서는 기업 고객(Account)이 다른 기업을 추천하는 구조입니다. 영업 담당자가 개입하거나, 자동화된 프로세스로 진행할 수 있습니다.

B2B vs B2C 차이점

항목B2CB2B
Advocate 유형개인 ContactAccount 또는 Contact (법인 담당자)
Friend 유형신규 개인 고객신규 기업 고객 (Lead → Account 전환)
추적 단위개인 행동 이벤트영업 기회(Opportunity) 단위
보상 유형바우처, 포인트리베이트, 크레딧, 서비스 혜택
연동 객체Contact, OrderAccount, Lead, Opportunity
영업 담당자 역할불필요Internal Organization Unit으로 추적
ℹ️
Sales Cloud 연동 필수B2B Referral Management는 Sales Cloud의 Lead·Opportunity 객체와 연동됩니다. MCE만으로는 구성이 불완전합니다.
