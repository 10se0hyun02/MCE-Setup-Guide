Marketing Cloud Next (MCN)

최종 업데이트: 2026.06.19 · 출처: Salesforce 공식 문서 (406페이지)

🚀 개요

Marketing Cloud Next(MCN)는 Salesforce 플랫폼 위에 네이티브로 구축된 차세대 마케팅 솔루션입니다. 기존 Marketing Cloud Engagement(MCE)가 별도 인프라로 운영되던 것과 달리, MCN은 Data 360(Data Cloud) + Experience Cloud + Flow Builder + Agentforce를 통합하여 단일 플랫폼에서 마케팅을 운영합니다.

ℹ️
MCE를 대체하는 것이 아닙니다MCN은 MCE의 대체제가 아니라 확장 레이어입니다. 기존 MCE 고객은 MCE를 유지하면서 MCN 기능을 점진적으로 도입하거나, 장기적으로 MCN으로 전환을 검토할 수 있습니다. Salesforce는 두 플랫폼을 병행 지원합니다.

핵심 차별점

  • 네이티브 Data 360 통합 — 별도 커넥터 없이 Data Cloud 고객 프로필에 직접 접근
  • Flow Builder 기반 자동화 — 개발자 없이 로우코드로 마케팅 자동화 구성
  • Agentforce 기본 탑재 — AI 에이전트가 마케팅 의사결정에 참여
  • Unified Messaging 내장 — 이메일·SMS·WhatsApp을 단일 채널로 관리
  • Real-time 세그멘테이션 — Data Cloud Segment 기반 즉각적인 대상 갱신

🔧 설정 절차

MCN 설정은 세 단계로 구분됩니다: 기본 설정 → 필수 설정 → 추가 설정

Phase 1 — 기본 설정 (Basic Settings)

  1. MCN 활성화 요청 — Salesforce 영업 담당자 또는 지원팀을 통해 MCN 라이선스 프로비저닝 요청
  2. Data 360 활성화 — Setup → Data Cloud Setup에서 Data 360 사용 설정 (약 15분 소요)
  3. 기본 이메일 채널 추가 — 지원 케이스를 통해 Default Email Channel 활성화 요청
  4. 데이터 보호 정보 입력 — 개인정보 처리 방침 URL, 데이터 보호 담당자 정보 설정

Phase 2 — 필수 설정 (Required Setup)

  1. Salesforce CRM Connector 생성 — Data 360과 CRM 간 데이터 동기화 설정
  2. 데이터 스페이스 선택 — 마케팅 데이터를 저장할 Data Space 지정
  3. Identity Resolution 설정 — 이메일·전화·쿠키 기반 고객 통합 규칙 구성
  4. Segment 생성 — 첫 번째 발송 대상 세그먼트 정의 및 테스트

Phase 3 — 추가 설정 (Additional Settings)

  1. Data Kit 설치 — 사용할 기능(이메일·SMS·WhatsApp 등)에 맞는 Data Kit 설치
  2. 브랜드 설정 — 로고, 기본 색상, 글꼴 등 브랜드 가이드라인 업로드
  3. 채널별 설정 — SMS 발신 번호, WhatsApp 번호 등 채널별 추가 구성
  4. Agentforce 설정 — Marketing Agent 활성화 및 Topic/Action 설정
⚠️
Data 360 먼저 설정해야 합니다Data 360이 활성화되지 않은 상태에서는 MCN의 대부분 기능을 사용할 수 없습니다. 모든 설정의 출발점은 Data 360 활성화입니다.

📦 Data Kit 설치

Data Kit은 채널별 기능을 Data 360에 연결하는 패키지입니다. 사용하려는 채널마다 해당 Data Kit을 설치해야 합니다.

📈 리드/컨택 스코어링

MCN은 Data 360과 Agentforce를 활용하여 리드(Lead)·컨택(Contact)·어카운트(Account)의 마케팅 참여도를 자동으로 점수화합니다.

스코어링 기반 데이터

  • 이메일 오픈·클릭·전환 이력
  • 웹사이트 방문 빈도·페이지뎁스
  • 폼 제출, 자료 다운로드 이벤트
  • 캠페인 참여 이력 (누적 기간 설정 가능)
  • Sales Cloud 활동 연동 (미팅, 통화 등)

스코어 활용

  • MQL(Marketing Qualified Lead) 자동 식별 — 임계점 이상 점수 획득 시 Sales Cloud로 자동 이관
  • 세그먼트 조건 활용 — 스코어 범위별 맞춤 캠페인 타겟팅
  • Agentforce 참고 데이터 — 에이전트가 고객 가치를 판단하는 데 활용

🔄 MCE에서 MCN으로 전환

ℹ️
즉시 전환이 아닌 점진적 도입을 권장Salesforce는 MCE에서 MCN으로의 빅뱅 전환보다 병행 운영을 통한 점진적 전환을 권장합니다. MCE의 기존 이메일 프로그램은 MCN 전환과 무관하게 계속 운영됩니다.

전환 시 주요 검토 항목

  • Data Extension → DMO 매핑 — 기존 DE 구조를 Data 360 DMO 스키마로 변환
  • Journey Builder → Flow Builder 재설계 — Journey의 로직을 Flow로 재구성 필요
  • AMPscript → Salesforce Formula 전환 — 동적 콘텐츠 로직 재작성
  • CloudPages → Experience Cloud 이전 — 랜딩 페이지·폼 재구성
  • 추적 코드 마이그레이션 — Collect JS → Interactions SDK 교체

권장 전환 순서

  1. Data 360 활성화 + Data Kit 설치 — 데이터 레이어 구축 선행
  2. 새 채널(SMS·WhatsApp) MCN으로 먼저 시작 — 리스크 낮은 신규 채널부터 MCN 방식 적용
  3. 세그먼트 병행 운영 — MCE Audience Builder와 Data Cloud Segment를 동시에 테스트
  4. 신규 캠페인부터 MCN 적용 — 기존 MCE 캠페인은 유지하고 신규 캠페인을 MCN으로 실행
  5. MCE 캠페인 순차 마이그레이션 — 성과가 낮거나 단순한 캠페인부터 순서대로 이전
