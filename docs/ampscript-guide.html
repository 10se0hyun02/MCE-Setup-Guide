AMPScript 실전 가이드

MCE 이메일·CloudPages에서 사용하는 서버 사이드 스크립트 언어. 변수·조건·DE 조회·동적 콘텐츠 생성 패턴을 실무 예제 중심으로 정리합니다.

AMPScript 개요

AMPScript는 Salesforce Marketing Cloud 전용 서버 사이드 스크립트입니다. 이메일 발송 시점에 서버에서 실행되어 수신자별로 다른 콘텐츠를 렌더링합니다. HTML 안에 %%[ ]%% 블록으로 삽입하며, 변수 치환은 %%변수명%% 형식입니다.

실행 환경

환경사용 위치주의사항

기본 문법 구조

AMPScript — 기본 구조
/* 1. 실행 블록: 로직 처리 (출력 없음) */ %%[ SET @변수명 = '값' SET @fullName = Concat(@firstName, ' ', @lastName) ]%% /* 2. 인라인 치환: 변수 값을 HTML에 직접 출력 */ 안녕하세요, %%=v(@fullName)=%% 님 /* 3. 인라인 함수 호출: =함수()= 형식 */ %%=Format(Now(), "yyyy-MM-dd")=%% /* 4. 제목줄(Subject)에서도 동일하게 사용 가능 */ %%=v(@firstName)=%% 님을 위한 특별 혜택
📌
v() vs 직접 치환: %%변수명%%도 작동하지만, %%=v(@변수명)=%%가 공식 권장 방식입니다. 특히 변수값에 특수문자가 포함될 경우 v()를 쓰세요.

변수 & 기본 함수

SET / VAR

SET은 변수에 값을 할당합니다. 변수명은 반드시 @로 시작합니다. VAR로 미리 선언만 할 수도 있습니다.

AMPScript — SET / VAR
%%[ /* 단순 값 할당 */ SET @name = 'Hong Gildong' SET @age = 30 SET @isVip = TRUE /* 함수 결과 할당 */ SET @today = Now() SET @upper = UpperCase(@name) /* DE 필드값 할당 (발송 대상 DE의 필드) */ SET @grade = AttributeValue('MemberGrade') SET @email = AttributeValue('EmailAddress') /* VAR — 선언만 (초기값 없음, 나중에 SET 가능) */ VAR @result ]%%
💡
AttributeValue() vs Lookup(): AttributeValue('필드명')은 발송 대상 DE의 해당 행 값을 그대로 가져옵니다. 다른 DE를 조회하려면 Lookup()을 사용하세요.

IIF — 인라인 조건

단순 참/거짓 분기를 한 줄로 처리합니다. IF 블록보다 가볍게 쓸 때 유용합니다.

AMPScript — IIF(조건, 참값, 거짓값)
%%[ SET @grade = AttributeValue('MemberGrade') /* 등급이 'VIP'면 '특별', 아니면 '일반' */ SET @label = IIF(@grade == 'VIP', '특별', '일반') /* 빈값이면 기본값 세팅 */ SET @name = IIF(Empty(AttributeValue('Name')), '고객', AttributeValue('Name')) /* 숫자 비교 */ SET @isAdult = IIF(AttributeValue('Age') >= 19, '성인', '미성년') ]%% %%=v(@label)=%% 고객님께 드리는 혜택

Empty() / IsNull()

NULL이거나 빈 문자열인지 확인합니다. DE에서 가져온 값이 비어 있을 때 기본값 처리에 필수입니다.

AMPScript — Empty / IsNull
%%[ SET @name = AttributeValue('Name') /* Empty(): NULL 또는 빈 문자열이면 TRUE */ IF Empty(@name) THEN SET @name = '고객' ENDIF /* IsNull(): 엄격한 NULL 체크 (빈 문자열은 FALSE) */ IF IsNull(@name) THEN SET @name = '고객' ENDIF /* 실무에서는 Empty()를 권장 — NULL과 빈값 모두 처리 */ ]%%
⚠️
NULL 미처리 위험: DE 필드값이 NULL인 채로 Concat()이나 수식에 사용하면 전체 결과가 빈값이 됩니다. 모든 DE 조회 후 반드시 Empty() 체크를 습관화하세요.

IF / ELSEIF / ELSE / ENDIF

다단계 조건 분기입니다. 등급·성별·지역 등 세그먼트별 다른 콘텐츠를 표시할 때 가장 많이 사용합니다.

AMPScript — IF 다중 분기 (회원 등급별)
%%[ SET @grade = AttributeValue('MemberGrade') ]%% %%[ IF @grade == 'VIP' THEN ]%%

VIP 전용 혜택: 30% 할인 쿠폰이 준비되어 있습니다.

%%[ ELSEIF @grade == 'GOLD' THEN ]%%

GOLD 회원 전용: 20% 할인 쿠폰을 드립니다.

%%[ ELSEIF @grade == 'SILVER' THEN ]%%

SILVER 회원 전용: 10% 할인 쿠폰을 드립니다.

%%[ ELSE ]%%

신규 회원 특별 혜택: 5% 할인 쿠폰을 드립니다.

%%[ ENDIF ]%%

비교 연산자

연산자의미예시
==같음 (대소문자 무시)@grade == 'VIP'
!=같지 않음@status != 'inactive'
> < >= <=숫자 비교@age >= 19
AND논리 AND@age >= 19 AND @grade == 'VIP'
OR논리 OR@grade == 'VIP' OR @grade == 'GOLD'
NOT부정NOT Empty(@name)
📌
HTML 태그 분리 패턴: %%[ IF ... THEN ]%%처럼 ]%%와 HTML을 분리하면 이메일 HTML 구조를 유지하면서 AMPScript 블록을 삽입할 수 있습니다. 이 패턴이 실무 표준입니다.

FOR / NEXT — 반복문

LookupRows()로 조회한 행 집합을 순회할 때 주로 사용합니다. 구매이력 목록, 장바구니 상품, 추천 상품 리스트 렌더링에 필수입니다.

AMPScript — FOR/NEXT 기본 구조
%%[ SET @rows = LookupRows('OrderHistory_DE', 'ContactKey', @contactKey) SET @rowCount = RowCount(@rows) ]%% %%[ IF @rowCount > 0 THEN ]%% %%[ FOR @i = 1 TO @rowCount DO ]%% %%[ SET @row = Row(@rows, @i) SET @orderNo = Field(@row, 'OrderNumber') SET @prodName = Field(@row, 'ProductName') SET @amount = Field(@row, 'Amount') ]%% %%[ NEXT @i ]%%
주문번호상품명금액
%%=v(@orderNo)=%% %%=v(@prodName)=%% %%=Format(@amount, 'N0')=%% 원
%%[ ELSE ]%%

최근 구매 내역이 없습니다.

%%[ ENDIF ]%%
⚠️
반복 성능 주의: FOR 루프 안에서 Lookup()을 다시 호출하면 행 수만큼 DE 조회가 발생합니다. 가능하면 루프 전에 필요한 데이터를 모두 조회해 두거나, 루프 내 조회를 최소화하세요.

Lookup — 단일 값 조회

DE에서 조건에 맞는 첫 번째 레코드의 특정 필드 값 하나를 반환합니다. 가장 자주 쓰는 DE 조회 함수입니다.

AMPScript — Lookup(DE명, 반환필드, 조건필드, 조건값)
%%[ SET @contactKey = AttributeValue('ContactKey') /* MemberInfo_DE에서 ContactKey가 일치하는 행의 Grade 필드 반환 */ SET @grade = Lookup('MemberInfo_DE', 'Grade', 'ContactKey', @contactKey) /* 조건이 2개인 경우 (다중 조건) */ SET @coupon = Lookup('Coupon_DE', 'CouponCode', 'ContactKey', @contactKey, 'IsUsed', 'false') /* 조회 결과가 없으면 NULL 반환 → Empty()로 체크 필수 */ IF Empty(@grade) THEN SET @grade = 'GENERAL' ENDIF ]%%
🚫
Lookup 결과가 여러 행인 경우: 조건에 맞는 행이 여러 개여도 Lookup은 첫 번째 행만 반환합니다. 여러 행을 순회해야 하면 LookupRows()를 사용하세요.

LookupRows / LookupOrderedRows

조건에 맞는 여러 행을 행 집합(Rowset)으로 반환합니다. RowCount()로 건수 확인, Row()로 개별 행 접근, Field()로 필드값 추출합니다.

AMPScript — LookupRows 핵심 함수 세트
%%[ SET @ck = AttributeValue('ContactKey') /* 전체 조회 */ SET @rows = LookupRows('Cart_DE', 'ContactKey', @ck) /* 정렬 조회: LookupOrderedRows(DE, 최대행수, 정렬필드, 정렬방향, 조건필드, 조건값) */ SET @recentRows = LookupOrderedRows('OrderHistory_DE', 3, 'OrderDate DESC', 'ContactKey', @ck) SET @cnt = RowCount(@rows) IF @cnt > 0 THEN FOR @i = 1 TO @cnt DO SET @row = Row(@rows, @i) SET @item = Field(@row, 'ProductName') SET @price = Field(@row, 'Price') SET @qty = Field(@row, 'Quantity') NEXT @i ENDIF ]%%
함수역할반환값
LookupRows(DE, 필드, 값)조건 일치 전체 행 조회Rowset
LookupOrderedRows(DE, N, 정렬, 필드, 값)정렬 후 최대 N행 조회Rowset
RowCount(rowset)행 수 반환Number
Row(rowset, index)index번째 행 반환 (1부터 시작)Row
Field(row, '필드명')행에서 필드 값 추출값 (타입 유지)

DE 수정 함수

CloudPages에서 폼 제출, 구독 처리 등 DE에 데이터를 쓸 때 사용합니다. 이메일 본문에서는 사용 불가 — CloudPages(Landing Page)에서만 실행됩니다.

AMPScript — DE 쓰기 함수 (CloudPages 전용)
%%[ /* UpsertDE: PK가 일치하면 UPDATE, 없으면 INSERT */ UpsertDE('Subscriber_DE', 1, /* PK 필드 수 */ 'ContactKey', @ck, /* PK */ 'EmailAddress', @email, 'OptIn', 'true', 'OptInDate', Now() ) /* InsertDE: 무조건 새 행 추가 */ InsertDE('FormLog_DE', 'ContactKey', @ck, 'SubmittedAt', Now(), 'FormData', @formData ) /* UpdateDE: 조건 일치 행 업데이트 */ UpdateDE('Member_DE', 1, 'ContactKey', @ck, 'IsActive', 'false', 'UnsubDate', Now() ) /* DeleteDE: 조건 일치 행 삭제 (신중히 사용) */ DeleteDE('TempQueue_DE', 1, 'JobID', @jobId) ]%%
🚫
이메일 본문에서 DE 쓰기 금지: 이메일 Preview나 Test Send 시에도 AMPScript가 실행되므로, 이메일 본문에 UpsertDE 등을 넣으면 테스트할 때마다 실제 DE에 데이터가 기록됩니다.

%%[ SET @email = AttributeValue('EmailAddress') SET @name = AttributeValue('Name') /* 이메일에서 도메인 추출 */ SET @atPos = IndexOf(@email, '@') SET @domain = Substring(@email, ADD(@atPos, 1), SUBTRACT(Length(@email), @atPos)) /* 이름이 빈값이면 이메일 앞부분을 이름으로 사용 */ IF Empty(@name) THEN SET @name = Substring(@email, 1, SUBTRACT(@atPos, 1)) ENDIF /* 전화번호 하이픈 제거 후 뒷 4자리 마스킹 */ SET @phone = AttributeValue('Phone') SET @masked = Concat(Substring(@phone, 1, 7), '-****') ]%%

AMPScript — 날짜 실전 예제
%%[ /* 오늘 날짜 (KST 기준, BU 타임존이 Asia/Seoul인 경우) */ SET @today = SystemDateToLocalDate(Now()) SET @todayStr = FormatDate(@today, 'YYYY년 MM월 DD일') /* 가입 후 경과 일수 */ SET @joinDate = AttributeValue('JoinDate') SET @daysSince = DateDiff('D', @joinDate, Now()) /* 쿠폰 만료일 (오늘 + 30일) */ SET @expiry = DateAdd(Now(), 30, 'D') SET @expiryStr = FormatDate(@expiry, 'YYYY-MM-DD') ]%% 이 쿠폰은 %%=v(@expiryStr)=%% 까지 사용 가능합니다.

실전 패턴 ① — 인삿말 개인화

수신자 이름을 개인화하되, 이름이 없을 때 안전하게 대체 처리합니다.

AMPScript — 인삿말 개인화 (NULL 처리 포함)
%%[ SET @firstName = AttributeValue('FirstName') SET @lastName = AttributeValue('LastName') /* 이름 조합: 성+이름, 둘 다 없으면 '고객' */ IF NOT Empty(@lastName) AND NOT Empty(@firstName) THEN SET @displayName = Concat(@lastName, @firstName) ELSEIF NOT Empty(@lastName) THEN SET @displayName = @lastName ELSEIF NOT Empty(@firstName) THEN SET @displayName = @firstName ELSE SET @displayName = '고객' ENDIF /* 발송 시간에 따른 인삿말 */ SET @hour = DatePart(SystemDateToLocalDate(Now()), 'H') IF @hour < 12 THEN SET @greeting = '좋은 아침입니다' ELSEIF @hour < 18 THEN SET @greeting = '안녕하세요' ELSE SET @greeting = '좋은 저녁입니다' ENDIF ]%%

%%=v(@greeting)=%%, %%=v(@displayName)=%% 님!

실전 패턴 ② — 등급별 콘텐츠 분기

회원 등급에 따라 다른 혜택 배너와 쿠폰을 보여주는 패턴입니다.

AMPScript — 등급별 쿠폰 & 배너 분기
%%[ SET @ck = AttributeValue('ContactKey') SET @grade = Lookup('MemberInfo_DE', 'Grade', 'ContactKey', @ck) IF Empty(@grade) THEN SET @grade = 'GENERAL' ENDIF /* 등급별 쿠폰 코드 조회 */ SET @coupon = Lookup('CouponPool_DE', 'CouponCode', 'Grade', @grade, 'IsIssued', 'false') /* 발급된 쿠폰 사용 처리 */ IF NOT Empty(@coupon) THEN UpdateDE('CouponPool_DE', 2, 'Grade', @grade, 'CouponCode', @coupon, 'IsIssued', 'true', 'IssuedTo', @ck, 'IssuedAt', Now()) ENDIF /* 등급별 할인율 설정 */ SET @discountRate = IIF(@grade == 'VIP', '30%', IIF(@grade == 'GOLD', '20%', IIF(@grade == 'SILVER', '10%', '5%'))) ]%%

%%=v(@displayName)=%% 님의 등급: %%=v(@grade)=%%

할인율: %%=v(@discountRate)=%%

%%[ IF NOT Empty(@coupon) THEN ]%%

쿠폰코드: %%=v(@coupon)=%%

%%[ ELSE ]%%

현재 사용 가능한 쿠폰이 없습니다.

%%[ ENDIF ]%%

실전 패턴 ③ — 구매이력 테이블 렌더링

최근 구매 3건을 이메일에 테이블로 표시합니다. LookupOrderedRows로 최신순 정렬 후 FOR 루프로 렌더링합니다.

AMPScript — 구매이력 테이블 (최근 3건)
%%[ SET @ck = AttributeValue('ContactKey') SET @rows = LookupOrderedRows('OrderHistory_DE', 3, 'OrderDate DESC', 'ContactKey', @ck) SET @cnt = RowCount(@rows) ]%% %%[ IF @cnt > 0 THEN ]%% %%[ FOR @i = 1 TO @cnt DO ]%% %%[ SET @row = Row(@rows, @i) SET @orderNo = Field(@row, 'OrderNumber') SET @prodName = Field(@row, 'ProductName') SET @amount = Field(@row, 'Amount') SET @status = Field(@row, 'OrderStatus') SET @bgColor = IIF(MOD(@i, 2) == 0, '#FAFAFA', '#FFFFFF') ]%% %%[ NEXT @i ]%%
주문번호 상품명 금액 상태
%%=v(@orderNo)=%% %%=v(@prodName)=%% %%=Format(@amount, 'N0')=%% 원 %%=v(@status)=%%
%%[ ELSE ]%%

최근 구매 내역이 없습니다.

%%[ ENDIF ]%%

실전 패턴 ④ — NULL 기본값 처리 템플릿

이메일 발송 전 필수 개인화 필드의 NULL 처리를 블록 상단에 한꺼번에 정의합니다. 대규모 이메일에서 가장 많이 발생하는 오류를 예방합니다.

AMPScript — 이메일 상단 NULL 처리 표준 블록
%%[ /* ── 기본 인적 사항 ── */ SET @name = AttributeValue('Name') SET @email = AttributeValue('EmailAddress') SET @grade = AttributeValue('MemberGrade') SET @phone = AttributeValue('Phone') /* NULL 기본값 처리 */ IF Empty(@name) THEN SET @name = '고객' ENDIF IF Empty(@grade) THEN SET @grade = 'GENERAL' ENDIF IF Empty(@phone) THEN SET @phone = '미등록' ENDIF /* ── 날짜 관련 ── */ SET @joinDate = AttributeValue('JoinDate') SET @lastPurchase = AttributeValue('LastPurchaseDate') IF Empty(@joinDate) THEN SET @joinDateStr = '정보 없음' ELSE SET @joinDateStr = FormatDate(@joinDate, 'YYYY-MM-DD') ENDIF IF Empty(@lastPurchase) THEN SET @daysSince = 0 ELSE SET @daysSince = DateDiff('D', @lastPurchase, Now()) ENDIF /* ── 금액 관련 ── */ SET @point = AttributeValue('AvailablePoint') IF Empty(@point) THEN SET @point = 0 ENDIF SET @pointStr = Concat(Format(@point, 'N0'), 'P') ]%%

이메일 링크에 subkey를 파라미터로 전달하면 CloudPages에서 수신자를 식별하고 DE를 업데이트할 수 있습니다.

AMPScript — 이메일 내 링크 (파라미터 포함)
/* 이메일 본문에서 CloudPage URL에 ContactKey 전달 */ %%[ SET @ck = AttributeValue('ContactKey') SET @grade = AttributeValue('MemberGrade') SET @pageUrl = Concat( 'https://pages.brand.co.kr/survey?', 'ck=', URLEncode(@ck), '&grade=', URLEncode(@grade) ) ]%% 설문 참여하기
AMPScript — CloudPages Landing Page (파라미터 수신 & DE 저장)
%%[ /* URL 파라미터 수신 */ SET @ck = QueryParameter('ck') SET @grade = QueryParameter('grade') SET @answer = QueryParameter('answer') /* 폼 제출값 */ /* 파라미터 유효성 검사 */ IF Empty(@ck) THEN Redirect('https://www.brand.co.kr/error') ENDIF /* 폼 제출 시 처리 (POST 방식) */ SET @method = RequestParameter('__RequestMethod') IF @method == 'POST' THEN SET @answer = RequestParameter('survey_answer') UpsertDE('SurveyResult_DE', 1, 'ContactKey', @ck, 'Answer', @answer, 'SubmittedAt', Now() ) SET @submitted = 'Y' ENDIF ]%% %%[ IF @submitted == 'Y' THEN ]%%

실전 패턴 ⑦ — 쿠폰코드 1:1 발급

쿠폰 풀 DE에서 미발급 쿠폰을 선착순으로 배정하고, 이메일에 수신자별 고유 쿠폰코드를 출력합니다. 중복 발급을 방지하려면 UpsertDE로 발급 상태를 즉시 업데이트합니다.

AMPScript — 1:1 쿠폰 발급 패턴
%%[ SET @ck = AttributeValue('ContactKey') /* 이미 발급된 쿠폰이 있는지 먼저 확인 */ SET @existingCoupon = Lookup('CouponIssue_DE', 'CouponCode', 'ContactKey', @ck) IF NOT Empty(@existingCoupon) THEN /* 재발송 방지: 기발급 쿠폰 재사용 */ SET @myCoupon = @existingCoupon ELSE /* 미발급 쿠폰 1건 조회 */ SET @newCoupon = Lookup('CouponPool_DE', 'CouponCode', 'IsIssued', 'false') IF NOT Empty(@newCoupon) THEN /* 발급 처리 (race condition 최소화를 위해 즉시 업데이트) */ UpdateDE('CouponPool_DE', 1, 'CouponCode', @newCoupon, 'IsIssued', 'true', 'IssuedTo', @ck, 'IssuedAt', Now()) InsertDE('CouponIssue_DE', 'ContactKey', @ck, 'CouponCode', @newCoupon, 'IssuedAt', Now()) SET @myCoupon = @newCoupon ELSE SET @myCoupon = '' /* 쿠폰 소진 */ ENDIF ENDIF ]%% %%[ IF NOT Empty(@myCoupon) THEN ]%%

회원님 전용 쿠폰코드: %%=v(@myCoupon)=%%

%%[ ELSE ]%%

죄송합니다. 쿠폰이 모두 소진되었습니다.

%%[ ENDIF ]%%
💡
대량 발송 시 Race Condition: 동시 발송 환경에서 Lookup → UpdateDE 사이에 동일 쿠폰이 중복 배정될 수 있습니다. 쿠폰 중복 방지가 매우 중요한 경우 Automation Studio로 사전에 ContactKey:CouponCode 매핑 DE를 만들어두고 이메일에서 조회만 하는 방식을 사용하세요.
