Mobile Studio

최종 업데이트: 2026.06.19  |  번역: MCE 구축 가이드 팀

개요

Mobile Studio는 모든 모바일 기기에서 개인화된 모바일 메시지로 고객에게 도달합니다. 세 가지 주요 채널을 제공합니다.

채널설명주요 용도
GroupConnect LINE·Facebook Messenger 등 메시징 앱을 통한 메시지 발송 LINE 팔로워 대상 마케팅, Journey 연동
MobileConnect SMS/MMS 문자 메시지 생성·발송·수신·추적 트랜잭션 알림, 마케팅 SMS, 키워드 자동응답
MobilePush 앱 사용을 유도하는 Push 알림 생성·발송 앱 재방문 유도, 프로모션, 트랜잭션 알림

GroupConnect (LINE)

GroupConnect는 메시징 앱을 통해 연락처에 메시지를 전송합니다. 현재 LINE 모바일 메시징 앱과 Facebook Messenger(Chat Messaging API 한정)를 지원합니다. LINE 앱은 Content Builder 및 Journey Builder와 통합되어 있습니다.

LINE 채널 설정

📌
사전 준비: LINE Developer Console에서 LINE Channel ID와 LINE Channel Secret을 확보하세요.
1
앱 스위처에서 이름 호버 → Setup
2
Quick Find에서 LINE 검색
3
Register New 클릭
4
LINE Channel ID 입력
5
채널 이름 입력
6
LINE Channel Secret 입력
7
Save
📌
채널 등록 후 메시지를 발송할 수 있으려면 최대 30분이 소요될 수 있습니다.

LINE 채널 수정 또는 비활성화

🚫
비활성화 전 필수 확인: 해당 채널을 사용하는 활성 Journey 또는 자동화가 없는지 반드시 확인하세요. 활성 Journey 또는 자동화가 있는 채널을 비활성화하면 오류가 발생하며, 비활성화된 채널은 재활성화가 불가능합니다.
1
Setup → Quick Find에서 GroupConnect 검색
2
이름·기타 값 수정: 채널 클릭 후 편집 (Channel ID는 편집 불가 — 실수 시 비활성화 후 재생성)
3
비활성화: 채널 옆 화살표 → Deactivate → 경고 메시지 확인

연락처 관리

GroupConnect의 Contacts 기능으로 연락처 정보를 조회·관리합니다. 시스템 테이블(GroupConnect LINE Demographics, GroupConnect LINE Subscriptions)에 연락처 정보가 저장되며, 이 테이블 데이터는 쿼리·내보내기가 불가능합니다.

연락처 추가 방법

  • Contact Builder에서 Contact Import 기능 사용 (FTP 위치에 연락처 파일 업로드)
  • Contacts API를 통한 프로그래밍 방식 추가

연락처 관련 주의사항

  • 하나의 LINE Address ID에 여러 ContactKey가 있으면 하나만 유지되고 나머지는 고아(orphaned) 처리됩니다.
  • 고아 ContactKey는 Import Clean-up에서 DataView(MobileLineOrphanContactView)로 확인 가능합니다.
  • ContactKey를 변경한 경우 메시지 발송 전 Journey 진입 데이터 익스텐션 업데이트 필수
  • GroupConnect LINE Contact import는 Update only 사용 권장

필터링 목록 생성

1
GroupConnect → Contacts → Lists → 목록 생성
2
속성 폴더에서 속성을 필터 섹션으로 드래그
3
필터 연산자와 값 설정 (예: ZIP Code equals 06100)
4
AND/OR 조건 설정
5
Save & Build → 이름 및 형식 선택 → 저장

자동 응답 메시지 설정

최대 50개 응답 규칙을 정의할 수 있습니다. GroupConnect는 수신 메시지에서 키워드를 식별하면 첫 번째로 일치하는 규칙의 응답을 발송하고 더 이상 탐색하지 않습니다.

💡
더 구체적인 규칙을 일반 규칙보다 우선순위 높게 설정하세요. GroupConnect는 수신 메시지와 첫 번째로 일치하는 규칙의 응답만 발송합니다.

MobileConnect (SMS / MMS)

MobileConnect로 SMS·MMS 문자 메시지를 생성·발송·수신·추적합니다. 드래그&드롭 인터페이스로 알림 및 트랜잭션 메시지를 발송하고, 수신 메시지 자동 응답, 키워드 관리 등 다양한 기능을 사용합니다.

MobileConnect 시작하기

1
연락처 목록 준비: CSV 파일에 최소 모바일 번호, 2자리 국가 코드, 고유 ContactKey(대개 모바일 번호) 포함. 이름·기타 필드 추가 가능
2
연락처 가져오기: CSV 파일을 MobileConnect로 가져오기. 그룹이 필요하면 가져올 때 목록 생성 가능
3
HELP 키워드 생성: 모든 MobileConnect 계정은 반드시 HELP 키워드를 보유해야 합니다 (관리자만 생성 가능)
4
기본 STOP 키워드 확인: STOP, QUIT, CANCEL, END, UNSUBSCRIBE는 즉각 전체 수신 거부 키워드로 자동 인식됨
5
메시지용 키워드 생성: 메시지용 키워드는 계정과 모바일 기기 간 SMS/MMS 교환에 사용되는 식별자
6
메시지 생성 및 발송: 메시지 템플릿 선택 후 생성·발송

SMS 코드 이해

코드 유형설명특징
Short Code 4-6자리 단축 번호 (예: 12345) 높은 발송량, 높은 신뢰도, 국가별 심사 필요
Long Code (10DLC) 일반 전화번호 형식 (미국 10자리) 낮은 발송량, Brand/Campaign 등록 필요
국제 Long Code 국제 트랜잭션 알림용 별도 웹폼·콜센터를 통한 수신 거부 옵션 제공 필요
⚠️
발송량·비용 최적화를 위해 국가 규정에 맞는 발신자 ID를 사용하세요. SMS 발송 전 해당 국가의 MobileConnect 가이드라인을 반드시 확인하고, 법적 요구사항에 대해서는 법률 자문을 받으세요.

키워드 관리

키워드는 계정과 모바일 기기 간 SMS/MMS 교환을 위한 식별자입니다. 키워드를 사용해 구독·수신거부·자동응답을 관리합니다.

키워드 유형설명
HELP모든 계정 필수. 구독자가 도움 요청 시 응답 (관리자 생성)
STOP 계열즉각 전체 수신 거부: STOP, QUIT, CANCEL, END, UNSUBSCRIBE (기본 인식)
커스텀 STOP관리자가 추가로 생성하는 수신 거부 키워드
메시지 키워드특정 SMS/MMS 교환을 위한 식별자 (메시지별 생성)

메시지 생성

MobileConnect에서 템플릿을 사용해 메시지를 생성합니다. Setup 앱에서 발신 차단 시간, 발신자 이름, 헤더·푸터, 구독 정보, 키워드, 단축/일반 코드 등 계정 레벨 설정을 관리합니다.

메시지 유형설명
Alert즉각적인 단방향 알림 발송
Keyword Interaction키워드 수신 시 자동 응답 시리즈
Conversation양방향 메시지 교환
Outbound Message (API)Journey Builder 또는 API를 통한 트리거 발송

연락처 가져오기 및 관리

MobileConnect에서 연락처를 가져오고 관리하여 올바른 오디언스에게 메시지를 발송합니다. 메시지 활동에서 성공 발송, 오류 발송, 기타 정보 요약을 확인할 수 있습니다.

MobilePush (앱 Push 알림)

MobilePush로 앱 사용을 유도하는 Push 알림을 생성·발송합니다. 앱에 MobilePush SDK를 통합하고, 위치 기반·타이머·이벤트 기반 알림을 지원합니다.

MobilePush 설정

1
Setup → MobilePush에서 앱 등록
2
iOS: APN(Apple Push Notification) 인증서 업로드
Android: Firebase (FCM) 서버 키 입력
3
앱에 MobilePush SDK 통합 (iOS/Android 개발자와 협업)
4
SDK에서 MobileConnect App Center Key 설정
5
테스트 기기 등록 및 Push 수신 확인
📌
MobilePush SDK는 iOS(Swift/Objective-C)와 Android(Java/Kotlin)를 지원합니다. React Native, Flutter 등 크로스플랫폼 SDK도 별도 제공됩니다.

Push 메시지 생성

메시지 유형설명트리거 방식
Outbound Message전체 또는 세그먼트 대상 일반 Push즉시 발송 또는 예약 발송
Location Entry Message지오펜스 진입 시 Push 발송위치 기반 트리거
Proximity Entry Message비콘 범위 진입 시 Push 발송비콘 기반 트리거
Inbox Message앱 내 메시지 함에 저장되는 메시지Journey 또는 직접 발송
💡
Push 알림 본문은 iOS 최대 4,096자(Payload 한도 내), Android는 실질적으로 제한 없습니다. 단, 디바이스 잠금화면에 표시되는 메시지는 약 50-100자 내외로 작성하는 것이 효과적입니다.
