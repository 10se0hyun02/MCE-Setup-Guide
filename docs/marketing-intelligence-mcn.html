Marketing Intelligence in Marketing Cloud Next

최종 업데이트: 2026.06.19  |  번역: MCE 구축 가이드 팀

개요

Marketing Intelligence in Marketing Cloud Next는 마케터가 여러 채널에서 대규모로 고객 경험을 개선할 수 있도록 지원합니다. 자동화된 커넥터, 통합 데이터 모델, 데이터 보강 기능을 통해 데이터 관리를 단순화하며, AI 기반 캠페인 요약 및 크로스채널 어트리뷰션으로 전환율과 캠페인 성과를 극대화합니다.

📌
Marketing Intelligence는 Data 360과 Tableau Next 라이선스가 있어야 사용할 수 있습니다. 에디션별 포함 서비스는 Salesforce 가격 문서를 확인하세요.
주요 기능설명
데이터 파이프라인광고 네트워크·애널리틱스·소셜 미디어 데이터를 자동 수집·정합
데이터 보강캠페인명 표준화, 비즈니스 컨텍스트 추가로 분석 정확도 향상
멀티터치 어트리뷰션이메일·SMS·소셜·검색 등 전채널 기여도 분석 (5가지 모델 제공)
Anchor Campaign관련 캠페인을 단일 성과 뷰로 통합 (Q4 프로모션 전체 성과 한눈에)
Agentforce 연동AI가 성과를 분석하고 예산 최적화·광고 일시정지 자동 추천

데이터 파이프라인

데이터 파이프라인은 마케팅 데이터의 수집·변환·정리를 자동화하여 Marketing Intelligence 분석을 위한 준비를 완료합니다. 사전 정의된 API 커넥터 또는 TotalConnect를 사용하며, 데이터 스트림·변환(transform) 등의 엔티티로 데이터를 구조화합니다.

파이프라인 핵심 구성 요소

구성 요소역할관계
Named Credential외부 시스템 인증 정보·엔드포인트 저장 (OAuth, Basic 인증 등)여러 Connection에서 재사용 가능
ConnectionData Cloud와 외부 소스 연결 — Named Credential 참조1개 Connection = 1개 Pipeline
Pipeline데이터 수집→변환→정합→Data Cloud 로드 전 과정 관리Named Credential에서 인증 획득
💡
재사용 패턴: Google Ads 계정 3개를 연결할 때, Named Credential은 1개만 생성하고 Connection·Pipeline을 계정별로 3개 만들면 됩니다. 인증 토큰이 만료되어도 Named Credential 1곳만 업데이트하면 모든 파이프라인에 자동 반영됩니다.

데이터 정합성 (Harmonize)

마케팅 데이터를 일관성 있고 구조화된 형태로 만들어 분석하기 쉽게 만드는 과정입니다. 패턴과 데이터 보강을 사용해 이름 표준화, 주요 속성 추출, 비즈니스 컨텍스트 추가로 오류를 줄이고 더 정확한 인사이트를 생성합니다.

데이터 보강 (Data Enrichment)

비즈니스 컨텍스트 추가, 일관성 향상, 분류 개선을 통해 마케팅 데이터의 품질과 활용도를 높입니다. Data 360 DMO, 업로드 파일, Einstein 데이터 보강 3가지 방식을 지원합니다.

보강 방식설명주요 사용 시나리오
Data 360 Objects (DMO)기존 Data 360 데이터 모델 오브젝트 활용표준화된 CRM 데이터 기반 보강
파일 업로드CSV 파일로 직접 데이터 업로드외부 참조 데이터 또는 수동 매핑
Einstein 보강AI가 데이터 정규화·그룹화 자동 수행facebook/FACEBOOK/FaceBook → Facebook 통일

캠페인명·광고명·광고 그룹명 등에 네이밍 컨벤션(패턴)을 적용하여 데이터를 더 구조화되고 분석하기 쉽게 만듭니다. 패턴을 정의하면 긴 텍스트 문자열에서 제품 유형, 지역, 사업부 같은 주요 속성을 추출할 수 있습니다.

어트리뷰션 (Attribution)

어트리뷰션은 어떤 마케팅 활동·채널·터치포인트가 리드 전환, 파이프라인 생성, 계약 체결 같은 비즈니스 성과에 기여했는지 파악하는 과정입니다. 어트리뷰션을 통해 예산 투자 결정을 최적화하고 마케팅 활동의 비즈니스 임팩트를 증명할 수 있습니다.

핵심 용어 정의

용어정의예시
인게이지먼트 신호사용자가 전환 전 브랜드와 가진 추적된 상호작용이메일 오픈, SMS 답장, 웹사이트 방문, 광고 클릭
전환 이벤트어트리뷰션이 평가하는 단일 성과 이벤트결제 완료, 문의 폼 제출, 뉴스레터 구독
전환 지표전환 이벤트를 수량화하는 지표총 구매 수, 리드 생성 수, 구독 가입 수
Lookback Window전환 인정 기간 — 이 기간 내 터치포인트만 기여도 인정30일 Lookback: 30일 이전 상호작용은 기여도 0%
Anchor Campaign크로스채널 캠페인들을 묶어 성과를 단일 뷰로 보는 상위 캠페인Q4 제품 출시 캠페인 (이메일+소셜+검색 통합)

멀티터치 어트리뷰션 (MTA)

단일 터치 모델이 1개 상호작용에 전체 기여도를 주는 것과 달리, MTA는 전체 고객 여정을 평가합니다. 단일 전환 이벤트를 정의하고 광범위한 인게이지먼트 신호 전반에 기여도를 분배합니다.

모델최적 사용 시나리오동작 방식예시
First Touch상위 퍼널 브랜드 인지도 분석첫 번째 터치포인트에 100% 기여도이메일 → Google Ads → 전환: 이메일 100%
Last Touch하위 퍼널 직접 전환 요인 최적화마지막 터치포인트에 100% 기여도이메일 → Google Ads → 전환: Google Ads 100%
Linear고객 생애 전체 상호작용 동일 가중모든 터치포인트에 균등 분배3개 터치포인트: 각 약 33%
Time Decay최근 상호작용 가중치 부여 전략전환에 가까울수록 높은 기여도전환 1일 전 광고 > 3일 전 이메일
U-Shaped첫 발견과 최종 전환을 동등하게 강조첫·마지막 터치 각 40%, 중간 합계 20%이메일:40%, Meta:10%, LinkedIn:10%, Google:40%

퍼널 기반 어트리뷰션

고객이 수익 퍼널의 정의된 단계(예: 이메일 발송 → 클릭 → 장바구니 추가 → 주문 완료)를 통해 어떻게 이동하는지 추적합니다. 멀티터치 어트리뷰션과 달리 상호작용 순서가 중요하며 사전 정의된 체크포인트 순서대로 추적합니다.

📌
퍼널 단계는 최대 5개까지 정의할 수 있으며, Draft 상태에서는 언제든지 수정·재정렬이 가능합니다. Deploy 후에는 수정 불가입니다.
